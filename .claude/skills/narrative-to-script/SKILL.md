# Narrative to Script — 叙事转影片解说文案

## 用途

将用户提供的叙事性文章（故事、小说、纪实文学、传记片段等）转化为**影片讲解风格的解说文案**。

## 使用方法

```
/narrative-to-script <原文内容或文件路径>
```

或直接在对话中提供叙事内容，Skill 会自动触发。

## 可选参数

- `时长` — 控制解说时长，如"8分钟"、"10分钟"
- `风格` — 解说风格偏好，如"悬疑紧张"、"温情治愈"、"犀利毒舌"
- `受众` — 目标观众，如"Z世代"、"文学爱好者"

## 输出格式

输出严格按以下结构：

1. **标题建议**（3个备选：悬念式/情感共鸣式/反常识观点式）
2. **解说正文**（含节奏标注，按时间段分段）
   - 开场钩子（0:00-0:30）
   - 叙事段落（每2-3分钟一个情绪小高潮）
   - 深度解析段落（主题解读/人物心理/时代隐喻）
   - 收尾观点（留白式结尾，不提供标准答案）
3. **解说备注**（配音情绪/BGM/视觉素材建议）

## 核心原则

1. **叙事脉络完整保留**：因果链清晰，人物动机可追溯，情感弧线不断裂
2. **独特评价与看法**：不是流水账复述，而是带有创作者个人视角的解读
3. **节奏控制**：每2-3分钟一个情绪小高潮或认知刷新点
4. **第一人称介入**：适度使用"我"建立创作者与观众的信任关系

## 质量控制

生成后自动自检：
- 观众仅凭解说能否理解完整因果链？
- 是否提出至少3个原文未明说的观点？
- 是否有"流水账"感？
- 结尾是否提供"带走的问题"而非"灌下的鸡汤"？
