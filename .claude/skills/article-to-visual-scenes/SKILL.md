# Article to Visual Scenes

将文章拆解为视频博客可用的**静态视觉场景** JSON，支持风格自定义。

> **重要原则：所有画面均为静态描述。**
> 本 skill 只生成静态分镜画面（类似传统 storyboard），不描述任何动画、运动、转场特效、粒子效果、镜头推拉等动态元素。`prompt` 字段专注于：画面里有什么、在什么位置、什么颜色、什么构图。

## 输出格式

```json
{
  "meta": {
    "title": "视频标题",
    "source": "源文件路径",
    "scene_count": 21,
    "style_summary": "画面风格概述（氛围、质感、视觉倾向）",
    "global_style_prompt": "全局画面描述（用于统一视觉基调）"
  },
  "prompts": [
    {
      "scene_id": 1,
      "title": "场景标题",
      "prompt": "静态画面描述（100-200字），包含背景、主体、构图、色彩、文字元素"
    }
  ]
}
```

## 工作流程

### 步骤 1：接收并分析文章

读取用户提供的文章/文案，分析其结构：

- 识别文章的整体结构（开头钩子、主体章节、结尾引导）
- 标记每个段落的语气标注（如文章中有的话，优先使用）
- 识别内容类型分布：
  - 概念解释（抽象理论、定义、特征）
  - 故事叙述（案例、寓言、叙事）
  - 历史回顾（时间线、人物、事件）
  - 引用展示（原文引述、金句）
  - 批判性分析（论证、反驳、对比）
  - 互动提问（向观众发问）
  - 总结（归纳、回顾）
  - 开头钩子 / 结尾引导

### 步骤 2：规划场景

基于分析结果，将文章拆解为 **15-25 个场景**。规划原则：

- 每个场景覆盖一个完整的讲解单元（一个概念、一个故事、一个论点）
- 场景数量与口播时长匹配（通常 10 分钟视频约 20 个场景）
- 场景之间要有逻辑衔接

### 步骤 3：生成 `meta` 对象

| 字段 | 说明 |
|------|------|
| `title` | 视频标题 |
| `source` | 源文章文件路径 |
| `scene_count` | 场景总数 |
| `style_summary` | 画面风格概述：整体氛围、视觉质感、色调倾向（如冷/暖、明/暗） |
| `global_style_prompt` | 全局画面基调：统一的整体视觉描述，如画面质感、明暗关系、构图倾向。不绑定具体配色或字体 |

### 步骤 4：生成 `prompts` 数组

每个 prompt 对象结构：

| 字段 | 说明 |
|------|------|
| `scene_id` | 场景编号，从 1 开始递增 |
| `title` | 场景标题（10-20 字），简洁概括画面内容 |
| `prompt` | **核心字段**：100-200 字静态画面描述。包含：背景环境、画面主体、元素位置、构图方式、色彩方案、文字叠加内容。禁止任何动画/运动/特效描述 |

### 步骤 5：输出

输出完整 JSON，保存到用户指定路径，或返回 JSON 内容。

## prompt 写作规范

### 默认风格：温暖治愈等距3D插画风

除非用户明确指定其他风格，否则所有场景默认采用 `warm-isometric` 风格。写作时必须体现以下特征：

- **等距透视(2.5D)**：所有物体采用固定角度的等距视角，呈现轻微立体感但保持扁平插画特征
- **暖色调背景**：米色、奶油色、浅棕色作为主基调，搭配复古纸张纹理
- **装饰性边框**：背景带有波浪纹、云纹或几何图案装饰
- **Q版微型人物**：场景中出现可爱的Q版小人（古人、农民、学者、乐师等），增加生活气息
- **圆润造型**：物体简化为圆润几何形状，边缘流畅，文字使用圆润字体
- **柔和投影**：阴影不硬，采用柔和的投影和环境光
- **微型场景叙事**：不是单一物体，而是构建完整的微型场景（桌面摆件、博物馆展台、古代村落等）
- **木牌/标牌**：文字信息通过木牌、标牌、竹简等可爱载体呈现

### 必须包含的要素

每个 `prompt` 必须覆盖以下要素：

1. **背景环境** — 画面的整体场景和底色（默认暖色调）
2. **画面主体** — 核心视觉元素是什么
3. **元素位置** — 各元素在画面中的空间关系（左/右/中/上/下）
4. **构图方式** — 分屏、居中、对称、留白等
5. **色彩倾向** — 主色调和明暗关系，不标注具体 HEX 色值
6. **文字叠加** — 画面上出现的文字内容，不涉及具体字体名称

### 禁止出现的描述

以下词汇/概念**禁止**出现在 `prompt` 中：

- **动画类**：动画、动态、运动、移动、滑动、旋转、缩放、推近、拉远、推进、淡入、淡出、闪烁、呼吸、抖动、粒子、消散、汇聚、爆裂、故障、glitch、扫描线、光流、缓缓、逐渐、慢慢、快速、突然
- **镜头类**：镜头、机位、视角、跟随、穿越、穿过、环绕、俯视、仰视、特写、全景、近景、远景、摇镜、移轴、Ken Burns
- **时间类**：然后、接着、之后、随即、约 X 秒后、X 秒后
- **特效类**：特效、光效、粒子效果、波纹、涟漪、3D 旋转、发光脉动

## 场景类型的静态画面策略

| 场景类型 | 画面策略 | 典型元素（等距3D风格） |
|---------|---------|---------|
| 开头钩子 | 强视觉冲击，信息密集 | 大字标题、核心符号、微型场景、Q版人物、悬念构图 |
| 概念解释 | 等距信息图表 + 清晰排版 | 几何图表、对比图、流程图、可爱图标、教具式布局 |
| 故事叙述 | 画面感强，有叙事层次 | 等距场景插画、Q版人物、情节定格、微型摆件 |
| 历史回顾 | 时间线 + 复古质感 | 等距时间轴、可爱文物模型、复古纸张、木牌标牌 |
| 引用展示 | 文字为主，仪式感 | 大字排版、圆润字体、柔和投影、木牌/竹简造型 |
| 批判性分析 | 对比 + 视觉冲突 | 分屏对比、等距场景并列、符号冲突 |
| 互动提问 | 开放式，留白多 | 简洁暖色背景、大字问题、可爱对话框、呼吸感 |
| 总结 | 信息图 + 回顾 | 核心符号排列、金句高亮、微型场景合集 |
| 结尾引导 | 温暖开放 | 暖色背景、Q版人物、可爱对话框、CTA按钮 |

## 风格预设（仅作概念参考）

以下风格预设用于帮助理解不同内容类型适合的画面氛围，但**不会输出具体的配色 HEX 值或字体名称到 JSON 中**。实际的风格设定（配色、字体、质感）由用户在后续生成图片时独立配置。

用户可通过以下方式指定风格倾向：

1. **选择预设名称**（快速）—— 如 `academic-tech`、`literary-warm`、`documentary-grey`
2. **提供风格描述文字**（自动解析）—— 如"我想要复古胶片质感"

### `warm-isometric` —— 温暖治愈等距3D插画风（默认）

适合：知识科普、文化内容、教育视频、社交媒体头图、儿童绘本

- **氛围**：温馨治愈、知识友好、可爱亲切
- **视觉倾向**：等距透视(2.5D)，扁平化+柔和立体阴影，米色/奶油色/浅棕色暖基调
- **质感**：复古纸张纹理、圆润几何造型、柔和投影、装饰性边框（波浪纹/云纹/几何图案）
- **典型元素**：Q版微型人物、微型场景摆件、木牌/标牌、可爱图标、桌面教具式布局

### `academic-tech` —— 学术科技风

适合：哲学思辨、学术解读、深度内容、思想史回顾

- **氛围**：理性、克制、信息密度高
- **视觉倾向**：几何线条和信息图为主，深色底配亮色点缀
- **质感**：简洁、现代、有纪录片感

### `literary-warm` —— 文学温暖风

适合：文学解读、散文、人物故事、温情叙事

- **氛围**：温暖、柔和、有人文气息
- **视觉倾向**：暖色调为主，复古纸张质感
- **质感**：插画感、手写笔触、暖光

### `documentary-grey` —— 纪实灰调风

适合：历史回顾、调查报道、严肃新闻、社会观察

- **氛围**：冷静、客观、有历史厚重感
- **视觉倾向**：黑白灰为主，偶现暖色点缀
- **质感**：档案照片、时间轴、文献质感

## 注意事项

- `prompt` 必须是**静态画面描述**（100-200 字），描述画面里有什么、在什么位置、什么构图。禁止出现任何动画、运动、镜头、转场特效描述
- **默认风格为 `warm-isometric`（温暖治愈等距3D插画风）**，除非用户明确指定其他风格，否则所有场景必须体现等距透视、暖色调、Q版人物、微型场景、圆润造型等特征
- `style_summary` 只描述整体氛围和视觉倾向（如"暖色调、温馨治愈、等距3D"），**不包含具体 HEX 色值或字体名称**
- `global_style_prompt` 是概括性的画面基调描述，**不绑定具体配色、质感参数或字体**，但必须包含等距视角和暖色调的描述
- `prompt` 中可描述颜色倾向（如"米色背景、暖棕色文字"），但**不标注 HEX 色值，不指定具体字体**
- 文字叠加要提炼**关键词**，不是完整句子，文字载体可以是木牌、标牌、竹简、气泡框等
- 场景数量建议 **15-25 个**，过少则覆盖不足，过多则过于碎片化
- 拒绝写实渲染和复杂3D透视，保持扁平插画+等距立体的"伪3D"效果
