# Book Reader — 交互式读书页面生成器

将已完成 `book-master` 工作流（含 `chapters/` 和 `reports/`）的书籍项目，生成为左右分栏的交互式 HTML 阅读页面。

## 使用方式

```
/book-reader /Users/xxx/learnAbook/有意识的心灵：一种基础理论研究
```

## 工作流程

### Step 1: 验证输入路径

- 路径必须是**绝对路径**
- 路径下必须存在 `chapters/` 和 `reports/` 目录
- `chapters/_index.md` 提供章节顺序和元信息

### Step 2: 提取章节与概念数据

对每章内容章节（排除 front-* 和 back-*）：

1. **读取原文**：`chapters/chapter-XX-标题.md`
2. **读取精读报告**：`reports/chapter-XX-标题-report.md`
3. **提取关键概念**：从报告"关键概念与定义"部分解析：
   - 术语名称（中英文）
   - 通俗解释
   - 文中定义
   - 关联概念
4. **原文概念标注**：在原文中定位概念出现的位置，添加 `<mark>` 标记

### Step 3: 生成 HTML 阅读页面

在 `{book-path}/reader/` 目录下生成：

```
reader/
├── index.html          # 入口页面：书籍概览 + 章节导航
├── chapter-01.html     # 第1章阅读页面
├── chapter-02.html     # 第2章阅读页面
├── ...
└── assets/
    ├── style.css       # 共享样式（如内联则省略）
    └── app.js          # 共享脚本（如内联则省略）
```

**页面布局（每章）**：

```
┌─────────────────────────────────────────────────────────────┐
│  📖 书名：有意识的心灵          [章1] [章2] [章3] ... [▓▓░░] │
├──────────────────────────────┬──────────────────────────────┤
│                              │  💡 本章概念                  │
│  📄 原文                      │  ┌────────────────────────┐  │
│                              │  │ 意识                     │  │
│  ...在某种意义上，意识的      │  │ 通俗解释：想象你正在...   │  │
│  附随性表达了一个非常自然     │  │ ──────────────────────── │  │
│  的思想。如果说B属性附随      │  │ 容易问题                 │  │
│  于A属性，那么不可能存在...   │  │ 通俗解释：想象你在看...   │  │
│                              │  └────────────────────────┘  │
│  [附随性] ← 高亮可点击        │                              │
│                              │  📌 点击左侧高亮词查看详情   │
│                              │                              │
├──────────────────────────────┴──────────────────────────────┤
│  ← 上一章                当前：第3章                下一章 → │
└─────────────────────────────────────────────────────────────┘
```

### Step 4: 概念匹配策略

**精确匹配**：概念名称直接出现在原文中
**模糊匹配**：概念的别名/英文出现在原文中
**段落关联**：如果某段落包含概念，该段落滚动时右侧对应概念卡片高亮

### Step 5: 输出验证

- 检查所有内容章节的 HTML 是否生成
- 检查概念提取数量是否合理（每章至少3个概念）
- 检查页面内链是否正常

## 页面交互功能

| 功能 | 说明 |
|------|------|
| **概念高亮** | 原文中的概念词语用彩色背景高亮 |
| **点击关联** | 点击高亮词 → 右侧滚动到对应概念卡片 |
| **悬浮提示** | 鼠标悬浮高亮词 → 显示通俗解释的 tooltip |
| **滚动联动** | 左侧滚动到某段落 → 右侧高亮该段落涉及的概念 |
| **章节导航** | 顶部章节按钮快速跳转，底部上一章/下一章 |
| **进度追踪** | 顶部进度条显示当前阅读位置 |
| **概念折叠** | 右侧概念卡片：默认显示通俗解释，点击展开完整定义 |

## 数据提取规范

### 从报告中提取概念

```markdown
### 概念N: {概念标题}
- **术语名称**: {名称} / {英文}
- **文中定义**: {定义文本}
- **解释**: {解释文本}
- **通俗解释**: {通俗解释文本}
- **关联概念**: {相关概念列表}
```

提取为 JSON：
```json
{
  "title": "概念标题",
  "term": "术语名称",
  "english": "英文名称",
  "definition": "文中定义",
  "explanation": "解释",
  "layman": "通俗解释",
  "related": ["关联概念1", "关联概念2"]
}
```

### 原文标注规则

1. 优先匹配术语名称中的中文部分
2. 其次匹配英文名称（大小写不敏感）
3. 避免过度标注：同一概念在同一段落内只标注首次出现
4. 嵌套概念处理：较长术语优先（如"意识的困难问题"优先于"困难问题"）

## HTML 页面规范

### 样式要求

- 左右分栏使用 CSS Grid：`grid-template-columns: 1fr 380px`
- 左侧原文区：`max-width: 720px`，`line-height: 1.8`，`font-size: 16px`
- 右侧概念区：`position: sticky; top: 0; height: 100vh; overflow-y: auto`
- 概念高亮：`background: linear-gradient(...)` 不同概念用不同柔和色彩
- 暗色模式支持：`@media (prefers-color-scheme: dark)`
- 响应式：移动端变为上下布局

### 性能要求

- 每章数据内嵌为 `<script type="application/json">` 避免外部请求
- CSS/JS 内联在 HTML 中（单文件即可打开）
- 大章节（>100KB 原文）采用虚拟滚动或懒加载

## 注意事项

- 书籍路径必须是绝对路径
- 报告中的通俗解释字段是显示重点，如缺失则使用"解释"或"定义"
- 章节顺序以 `chapters/_index.md` 中的表格为准
- 生成的 HTML 文件可以双击在浏览器中直接打开
- 如某章缺少对应报告，跳过概念提取，只显示原文
