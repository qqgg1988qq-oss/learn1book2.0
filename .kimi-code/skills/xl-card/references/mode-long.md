# 模具：长文卡（-l / 默认）

## 步骤 1：内容预处理

- 识别标题候选（`#`/`##` 开头、独立短行、或文章开头第一句核心观点）
- 识别金句候选（独立成段的短句，< 25 字，承载核心洞察）
- 识别条目候选（有"标题+解释"结构的内容，如概念定义、要点列表）
- 识别行动提示（"怎么做"类句子、建议、CTA）
- 按空行分割为段落列表

## 步骤 2：结构设计

按以下优先级排布：

1. **主标题**（必填）：从标题候选中选最有冲击力的一句，或从金句中提炼。≤20 字。
2. **副标题**（选填）：需要补充语境时使用。如主标题是设问，副标题可以是暗示答案的一句话。≤30 字。
3. **金句高亮**（必有）：整张卡最核心的一句话。截出来能独立当微头条发。放在正文之前。
4. **正文**（3-5 段）：支撑金句的论证或叙述。不是金句的解释说明，而是让金句"立起来"的内容。
5. **条目组**（选填）：有多个并列要点时使用。每个条目 = 标题（≤8 字）+ 正文（1-2 句）。
6. **行动提示**（选填）：引导读者下一步做什么。不是"总结一下"，是"去做这个"。
7. **署名行**：八零后阿力的觉醒日记 · 来源

## 步骤 3：文案填充

用预处理提取的内容填充上述结构。规则：

- **主标题不要用原文标题**：原文标题通常是描述性的，卡片主标题需要是观点性的或悬念性的
- **金句不是摘要**：金句是全文最有情绪冲击力或认知颠覆力的一句话，不一定是"中心思想"
- **正文压缩率 70-80%**：原文 1000 字 → 卡片正文 200-350 字
- **条目最多 5 个**：超过 5 个要合并或删减
- **行动提示一句话**：不做解释，不给理由，只给动作

## 步骤 4：品味校验

按 `references/taste.md` 逐项检查，重点：禁用词、假数据、标题力度、正文节奏。

## 步骤 5：输出

按 SKILL.md 中的 JSON Schema 输出。长文卡 JSON 示例：

```json
{
  "type": "long",
  "title":    { "text": "≤20字观点性标题", "maxChars": 20, "role": "primary-heading" },
  "subtitle": { "text": "≤30字语境补充", "maxChars": 30, "role": "secondary-heading" },
  "highlight":{ "text": "最有冲击力的金句", "style": "quote-bar" },
  "body":     ["段落1", "段落2", "段落3"],
  "items":    [{ "label": "≤8字", "text": "1-2句" }],
  "action":   { "text": "一句动作引导" },
  "footer":   { "signature": "八零后阿力的觉醒日记", "source": "Dan Koe" }
}
```
