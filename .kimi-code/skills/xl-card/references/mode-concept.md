# 模具：概念卡（-c）

将单个概念（术语、理论、模型）结构化为一张文案卡片。

## 步骤 1：提取概念信息

从内容中提取：
- **概念名称**：术语或理论名
- **定义**：最简洁的学术/标准定义
- **通俗解释**：用大白话重说一遍
- **类比**：用一个日常场景解释这个概念
- **三个要点**：这个概念最重要的三个推论/应用/注意事项

## 步骤 2：结构设计

按以下顺序排布：

1. **主标题**：概念名本身（≤10 字）
2. **副标题**：一句话定义（≤25 字）
3. **金句高亮**：通俗解释中最精辟的那句大白话
4. **正文**：类比段落（1-2 句描述日常场景）
5. **条目组**：三个要点（每个 1-2 句）
6. **署名行**：八零后阿力的觉醒日记 · 来源

## 步骤 3：文案规则

- **类比必须能用一句话讲给大妈听**：不能用专业领域内的类比解释专业概念
- **三个要点递进**：不是并列，是"知道→理解→应用"的递进
- **不用括号加注**：正文中不出现"（即……）""（又称……）"

## 步骤 4：品味校验

按 `references/taste.md` 逐项检查。

## 步骤 5：输出

按 SKILL.md 中的 JSON Schema 输出。概念卡 JSON 示例：

```json
{
  "type": "concept",
  "title":     { "text": "概念名称", "maxChars": 10, "role": "primary-heading" },
  "subtitle":  { "text": "一句话定义", "maxChars": 25, "role": "secondary-heading" },
  "highlight": { "text": "大白话金句", "style": "quote-bar" },
  "body":      ["日常生活类比（1-2句）"],
  "items": [
    { "label": "≤8字要点1", "text": "知道层面" },
    { "label": "≤8字要点2", "text": "理解层面" },
    { "label": "≤8字要点3", "text": "应用层面" }
  ],
  "footer":    { "signature": "八零后阿力的觉醒日记", "source": "Dan Koe" }
}
```
