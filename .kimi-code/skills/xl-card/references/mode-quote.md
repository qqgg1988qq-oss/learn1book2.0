# 模具：金句卡（-q）

将一句/一段话铸成独立可传播的文案卡片。

## 步骤 1：提取金句

从内容中识别：
- 独立成段的短句（< 30 字）
- 承载核心洞察或情绪冲击力
- 截出来能独立传播（不需要上下文就能理解）

如果内容中没有天然金句，可以**提炼**一句——基于内容的核心观点，重写为一句短小有力的话。

## 步骤 2：结构设计

1. **主标题**（大字位）：金句本身（≤30 字）
2. **副标题**（选填）：出处/作者/语境（如"——《大器晚成》第5章"）
3. **正文**（选填）：一句解读或延伸（≤50 字）。不是"这句话的意思是……"，而是让金句再多一层的追问或画面。
4. **署名行**：八零后阿力的觉醒日记 · 来源

## 步骤 3：文案规则

- **金句本身不改动**：如果是原文引用，一字不改
- **解读要加"一层"不是"解释"**：好的解读不是翻译金句，是让读者看完金句后"嗯……"、看完解读后"哦！"
- **总字数 ≤100 字**

## 步骤 4：品味校验

按 `references/taste.md` 逐项检查。

## 步骤 5：输出

按 SKILL.md 中的 JSON Schema 输出。金句卡 JSON 示例：

```json
{
  "type": "quote",
  "title":    { "text": "金句原文一字不改", "maxChars": 30, "role": "primary-heading" },
  "subtitle": { "text": "——《出处》作者", "maxChars": 30, "role": "secondary-heading" },
  "body":     ["一句延伸解读，加一层追问而非解释"],
  "footer":   { "signature": "八零后阿力的觉醒日记", "source": "Dan Koe" }
}
```
