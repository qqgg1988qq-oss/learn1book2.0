## 工作流程

### Step 1: 读取并分析文章内容

读取用户提供的文章/主题，提取：
- **核心主题**：文章在讲什么
- **目标人群**：谁会读这篇文章
- **核心利益**：读者能得到什么
- **情感基调**：正面/负面/中性？焦虑/希望/愤怒？
- **领域分类**：社会热点 / 情感生活 / 历史文化 / 科技财经 / 健康养生

### Step 2: 读取并匹配痛点图谱

1. **读取** `/Users/chouchou/Documents/Obsidian Vault/九局下半怎么破局/参考资料/40岁中年危机痛点图谱_完整报告.md`
2. 将文章主题、核心论点、关键数据与图谱中的20个痛点类型逐一比对
3. 按**匹配程度**排序，标注每个痛点的匹配理由
4. 输出**至少3个高匹配痛点**，作为标题创作的主要方向

**匹配原则**：
- 文章直接讨论该痛点 → 高匹配
- 文章涉及该痛点的上游原因或下游后果 → 中高匹配
- 文章主题与该痛点的目标人群高度重合 → 中匹配
- 仅有弱关联 → 低匹配（可忽略）

### Step 3: 确定心理驱动

在已匹配的痛点基础上，选择核心心理触发机制：

| 领域 | 最强心理驱动 | 触发关键词 |
|------|------------|-----------|
| 社会热点 | 愤怒/正义(20%) + 好奇(20%) | 质问、不公、揭秘、最新 |
| 情感生活 | 好奇(37%) + 共鸣(13%) | 为什么、扎心、崩溃、真相 |
| 历史文化 | 好奇(82%) | 揭秘、不为人知、真相、竟 |
| 科技财经 | 贪婪(67%) | 月入、赚钱、加薪、翻倍 |
| 健康养生 | 恐惧(24%) + 好奇(21%) | 千万别、险送命、别吃、远离 |

### Step 4: 选择主公式 + 叠加辅助技巧

**主公式选择**（从10大公式中选1-2个）：

| # | 公式名 | 模板 | 适用场景 |
|---|-------|------|---------|
| 1 | 数字悬念利益 | `[数字]招[动作]，[利益结果]！[权威]` | 健康/职场/副业 |
| 2 | 反常识颠覆 | `[普遍认知]？[否定]，[反常识结论]` | 科普/历史/健康 |
| 3 | 揭秘真相 | `[揭秘][事件]，[不为人知]的[真相]` | 历史/军事/社会 |
| 4 | 故事数字反转 | `[年龄/身份]+[数字]，[事件]+[反转]` | 情感/家庭/故事 |
| 5 | 痛点利益 | `[痛点场景]，[利益承诺/解决方案]` | 健康/职场/育儿 |
| 6 | 冲突反转爽感 | `[压迫]，[反击]，[爽感结局]` | 职场/家庭冲突 |
| 7 | 设问引导 | `[为什么/如何][现象]？[悬念]` | 情感/育儿/科普 |
| 8 | 热点绑定 | `[热点事件]，[利益/悬念]` | 科技/财经/社会 |
| 9 | 权威揭秘 | `[身份/数据]，[揭秘]+[内容]` | 健康/职场/财经 |
| 10 | 极端情绪 | `[极端情绪/行为]，[反差结果]` | 情感冲突/正能量 |

**辅助技巧叠加**（选1-2个）：
- +数字 → 增强可信度
- +悬念 → 用"它""这一步""这样"不透露答案
- +权威 → 老中医/10年HR/央行数据
- +热点 → 绑定DeepSeek/AI/最新政策
- +冲突 → 制造反差和矛盾
- +情感 → 扎心/泪崩/暖心

### Step 5: 生成候选标题（至少10个）

**输出要求**：
- 生成 **至少10个** 候选标题
- 标题必须**基于 Step 2 匹配的痛点类型**创作
- 覆盖不同公式和技巧组合
- 每个标题标注使用的**痛点类型**、**公式编号**和**技巧标签**
- 字数控制在 **16-25字**（最佳区间）
- 前13字必须包含至少1个钩子
- 最终必须保存到文件（见 Step 8）

**标题格式**：
```
候选1: [标题内容] | 痛点：[痛点类型] | 公式X+技巧Y+技巧Z | 预估点击率:高/中
```

### Step 6: 按痛点匹配度排序并推荐TOP3

- 优先推荐与高匹配痛点结合的标题
- 每个推荐标题说明其匹配的痛点和推荐理由
- TOP3 应覆盖不同痛点或不同情绪角度

### Step 7: 自检与优化

对每条标题进行以下检查：

| 检查项 | 合格标准 |
|--------|---------|
| 字数 | 16-25字最佳，12-28字可接受 |
| 前13字 | 包含数字/悬念/冲突/利益之一 |
| 数字 | 有具体数字优先（3-7最佳） |
| 悬念 | 话说一半，有信息缺口 |
| 情绪 | 能调动至少一种情绪 |
| 利益 | 读者知道"看了能得到什么" |
| 合规 | 不涉及违法违规、过度夸大 |
| 痛点匹配 | 标题与高匹配痛点直接相关 |

### Step 8: 推荐标签（可选）

当用户需要为文章添加头条号标签/话题时，执行此步骤。

**标签文件输出规则**：
- 无论用户是否明确说"保存"，最终都必须将标题和标签结果保存到文件
- 文件保存位置：**输入文件所在目录**
- 文件命名：`{输入文件名（去扩展名）}_titles.md`
- 例如输入为 `chapter-09-script.md`，则输出 `chapter-09-script_titles.md`
- 若输入为纯文本而非文件，则文件名取文章前10字或主题词 + `_titles.md`
