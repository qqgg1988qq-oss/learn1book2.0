# Deep Reader — 文章深度精读与知识点提取

对给定文章进行**深度精读**，**完整、无遗漏**地提取所有知识点，按10个维度结构化输出。

## 使用方式

### 方式一：直接传入文本

```
/deep-reader
[粘贴文章全文]
```

### 方式二：读取文件

```
/deep-reader ./article.md
```

### 方式三：读取已拆分的章节

```
/deep-reader ./book-chapters/chapter-01-xxx.md
```

## 工作流程

1. **获取文章**：从参数中提取文本内容，或读取指定文件
2. **完整阅读**：通读全文，标记关键信息
3. **逐维度分析**：按10个维度系统提取
4. **自检校验**：对照自检清单，查漏补缺
5. **输出报告**：按固定格式输出结构化报告

## 分析维度（必须全部覆盖）

### 一、元信息
- 文章标题、作者、出处
- 文章类型
- 核心主题与写作目的

### 二、核心论点
- 中心论点与分论点
- 论点间的逻辑关系

### 三、关键概念与定义
- 所有专业术语、新概念
- 术语名称、文中定义、解释、关联
- **通俗解释**：用日常语言、比喻或类比，让一个完全不懂该领域的读者也能理解这个概念

### 四、论据与证据
- 数据、案例、引用、修辞
- 标注每条论据支撑的论点

### 五、逻辑结构与推理链
- 总体结构
- 段间衔接
- 关键推理步骤

### 六、方法论
- 研究方法、分析框架
- 技术、工具、流程
- 优势与局限

### 七、隐含知识与前提假设
- 默认背景知识
- 未经证明的前提
- 学术/行业背景

### 八、重要引用原文
- 3-10句核心原文
- 标注重要性

### 九、知识网络与关联
- 与外部领域知识的关联
- 可延伸的主题
- 可联系的理论概念

### 十、批判性思考
- 论证薄弱环节
- 潜在反驳角度
- 知识边界

## 输出格式

使用固定格式输出，见 `template.md`。必须包含最后的**自检清单**。

## 注意事项

- 不得因知识点"不重要"而跳过
- 多层次解析：字面→深层逻辑
- 常识性概念在文中有特殊用法也需提取
- **关键概念必须附带通俗解释**：每个专业术语都要用比喻、类比或日常语言解释，让零基础读者也能理解
- 输出必须完整，不可省略任何维度
