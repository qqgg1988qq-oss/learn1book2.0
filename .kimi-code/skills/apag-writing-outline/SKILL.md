# APAG 写作大纲设计器

## 何时使用

当你需要把一份资料、一个主题或一个观点，转化为有说服力的长篇/短篇内容时使用。

适合的内容形式：
- Newsletter / 公众号文章
- 销售页 / 课程推广页
- 口播脚本 / 视频文案
- 推特/X  thread、LinkedIn 帖子
- 小红书/头条引流帖

## 工作流程

1. **读取用户资料**：先读取用户提供的文件或粘贴的文本。
2. **提取核心信息**：用一句话概括主题、核心观点、目标读者、Desired Outcome。
3. **按 APAG 四步设计大纲**：
   - **Attention（钩子/标题）**：先写出 3-5 个候选钩子，最后从正文中提炼最佳钩子。
   - **Perspective（视角/问题）**：描绘读者当前的问题、痛苦、错误信念。
   - **Advantage（优势/解决方案）**：提出更优视角、核心观点、社会证明、引用、案例。
   - **Amplify/Gamify（放大/游戏化）**：把转变拆解为 3-5 个可执行步骤，给出明确 CTA。
4. **输出结构化大纲**：按 `template.md` 的结构生成大纲。
5. **保存为 Markdown 文件**：将生成的大纲保存到**源文件所在目录**，文件名为 `{源文件名}-apag-outline.md`。
   - 例如：源文件为 `article.md`，则输出 `article-apag-outline.md`。
   - 如果源文件没有 `.md` 扩展名（如 `.txt`、`.pdf` 等），在源文件名后追加 `-apag-outline.md`。
   - 保存后向用户报告文件路径。

## APAG 四段说明

### Attention — 注意/钩子

- 功能是制造「信息鸿沟」，让读者产生好奇心。
- 常用元素：问题、好处、数字、时间框架、负面经历、反直觉观点。
- 评估标准：
  - **Relevance（相关性）**：与目标读者日常生活有多相关？
  - **Awareness（认知水平）**：复杂度是否匹配目标读者的认知水平？
  - **Effort（获取成本）**：读者能多快得到结果？
- **关键原则**：先写正文，最后写钩子。

### Perspective — 视角/问题

- 核心是「描绘问题的图景」，让读者感到被理解。
- 不要立刻给答案，先展示：
  - 读者当前的错误信念是什么？
  - 这个信念带来了什么痛苦？
  - 常见的失败模式有哪些？
- 目标：让读者在镜子前看清自己「衣服穿反了」。

### Advantage — 优势/解决方案

- 核心是「销售一个更好的视角/做事方式」。
- 包含：
  - 新颖观点 / 核心概念
  - 之前-之后（Before-After）对比
  - 社会证明：引用、案例、客户经验、数据
  - 个人经验 / 故事
- 目标：让读者相信「存在一条更近、风景更好的路」。

### Amplify/Gamify — 放大/游戏化

- 核心是「把转变拆解为可执行的阶梯」。
- 包含：
  - 3-5 个具体步骤
  - 每个步骤的即时收益
  - 降低行动摩擦的小技巧
  - 明确的 CTA（下一步做什么）
- 目标：像游戏任务树一样，让读者从「知道」走到「做到」。

## 输出格式

1. 先按 `template.md` 中的结构生成完整大纲内容。
2. 再将大纲内容保存为 Markdown 文件，路径为**源文件所在目录**，文件名为 `{源文件名}-apag-outline.md`。
3. 最后向用户展示文件保存路径，并附上大纲核心摘要或关键亮点。

## 注意事项

- 如果用户没有提供资料，先询问主题或请用户粘贴内容。
- 钩子要最后写，从正文中最有冲击力的部分提炼。
- 每个阶段尽量给出 2-3 个可选方案，方便用户挑选。
- **必须保存输出**：生成的大纲不要只展示在对话中，必须写入源文件同目录下的 `{源文件名}-apag-outline.md`。
- 保存前检查目标目录是否存在，不存在则创建。
- 输出保持中文，除非用户明确要求英文。
