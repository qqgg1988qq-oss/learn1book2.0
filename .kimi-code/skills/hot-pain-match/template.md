## 单平台匹配模板

### 输入示例

```
用户: "用头条热搜匹配痛点"
```

### 执行步骤

1. **加载配置 + 获取热搜**
   ```bash
   source ~/.claude/skills/tophub-trends/scripts/config.env
   curl -s --header "Authorization: $TOPHUB_ACCESS_KEY" \
     "https://api.tophubdata.com/nodes/x9ozB4KoXb"
   ```

2. **提取热搜标题列表**
   从 `data.items[]` 提取 `title`、`description`、`url`、`extra`

3. **逐条匹配 20 个痛点**
   对每条热搜，遍历 20 个痛点关键词表，计算创作价值分

4. **排序 + 输出**

## 全网榜中榜匹配模板

### 输入示例

```
用户: "今天的热搜和中年危机有什么关系"
```

### 执行步骤

1. 调用榜中榜接口：
   ```bash
   curl -s --header "Authorization: $TOPHUB_ACCESS_KEY" \
     "https://api.tophubdata.com/hot?date=$(date +%Y-%m-%d)"
   ```

2. 提取前 50-100 条热搜

3. 逐条匹配 20 个痛点

4. 排序输出（记录来源平台）

## 内容切入点生成模板

针对每条高价值匹配热搜，用以下框架生成切入点建议：

### APAG 微框架（一句话版）

```
A（钩子）: 热搜本身就制造了认知冲突
P（痛点）: 直接关联痛点，如"80%的中年人正在经历这个"
A（视角）: 文章/产品的独特视角是什么
G（放大）: 引导原文/产品/服务
```

### 示例

```
热搜: "35岁程序员被裁员后送外卖"
→ 匹配: #1 年龄歧视 ~80% + #20 副业失败 ~80%
→ 内容切入点: 
  "35岁被裁送外卖不是退路，是陷阱——80%的中年副业者收入不如原来。
   真正该做的不是多打一份工，是[文章核心观点]"
```
