# Audio2Text 使用示例

## 示例 1：基础转写

### 输入

```
/audio2text ./meeting.wav
```

### 处理流程

1. 读取 `./meeting.wav`，获取文件大小
2. 生成时间戳和随机串，计算 HMAC-SHA1 签名
3. POST 上传音频到 `/v2/upload`
4. 收到响应：`orderId = DKHJQ20250928163000ABC123`
5. 轮询 `/v2/getResult`，status=3（处理中）
6. 约 30 秒后，status=4（完成）
7. 解析 orderResult 中的 lattice 数据

### 输出

```
[0ms -> 1680ms] 喂你好，请问是王先生吗？
[2390ms -> 3640ms] 是的，请问有什么事情？
[5130ms -> 7200ms] 我是上次联系过您的张经理。
[7200ms -> 8650ms] 哦，张经理您好。
[9330ms -> 11240ms] 听说得到您有什么新消息吗？
```

## 示例 2：会议录音 + 角色分离

### 输入

```
/audio2text ./team-meeting.mp3 --role-type 1 --role-num 3
```

### 输出

```
[Role 1 | 0ms -> 3200ms] 大家好，今天的会议由我来主持。
[Role 2 | 3400ms -> 5800ms] 好的，我先汇报一下上周的进度。
[Role 3 | 6000ms -> 7200ms] 我这边有个问题需要讨论。
[Role 1 | 7400ms -> 8900ms] 好的，一个一个来，先说进度。
```

## 示例 3：方言音频

### 输入

```
/audio2text ./dialect-interview.mp3 --language autodialect --pd general
```

### 说明

`autodialect` 支持中英混说及 202 种中文方言免切换识别，无需手动指定方言类型。

## 示例 4：视频文件转字幕

### 输入

## 常见错误及处理

### 错误码 100008

**原因**：请求时间戳与服务器时间偏差过大  
**解决**：同步系统时钟：`sudo ntpdate -u time.apple.com`

### 错误码 100009

**原因**：签名校验失败  
**解决**：检查 API_SECRET 是否正确；确认参数编码方式（需对 key 和 value 分别 URL 编码）

### 错误码 100003

**原因**：dateTime 格式错误  
**解决**：格式必须为 `yyyy-MM-dd'T'HH:mm:ss±HHmm`，如 `2025-09-08T22:58:29+0800`

### 状态码 -1（订单失败）

**可能原因**：
- failType=4：音频时长超限（>5小时）
- failType=5：duration 参数与实际时长不符
- failType=6：静音文件，无有效语音

## API 响应状态对照

| status | 含义 | 操作 |
|--------|------|------|
| 0 | 订单已创建 | 等待处理 |
| 3 | 处理中 | 继续轮询 |
| 4 | 已完成 | 获取结果 |
| -1 | 订单失败 | 查看 failType |
