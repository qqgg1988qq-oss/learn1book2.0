# Batch Knowledge Extractor — 批量文件深度精读与知识整合

将多个文件（PDF、Markdown、TXT 等）通过**文本提取 → 并行深度精读 → 整合输出**的流水线，自动完成批量知识提取与整合。

## 工作流概览

```
用户输入（文件列表 / 文件夹路径）
    ↓
[Step 1] 文本提取 — 检测文件类型，PDF用OCR，其他直接读取
    ↓
[Step 2] 并行深度精读 — 每个文件启动一个子代理，按10维度分析
    ↓
[Step 3] 知识整合 — 读取所有报告，整合为总报告
    ↓
输出：每文件一份精读报告 + 一份整合总报告
```

## Step 1 — 文本提取

### 1.1 检测文件类型

| 文件类型 | 处理方式 | 工具 |
|----------|----------|------|
| `.md` / `.txt` | 直接读取 | Read 工具 |
| `.pdf` | 先尝试直接提取文本；若无文本层，用OCR | `scripts/extract_pdf_text.py` |
| `.srt` | 清洗时间轴与标记，合并为连续文本 | `scripts/srt_to_md.py` |
| 其他 | 尝试读取，若失败则告知用户 | — |

### 1.2 PDF OCR 提取

使用脚本 `scripts/extract_pdf_text.py`：

```bash
python3 scripts/extract_pdf_text.py <pdf_path_or_folder> [output_dir] [lang]
```

参数：
- `lang`: 默认 `chi_sim+eng`（中文+英文），纯英文可用 `eng`

依赖：
- `PyMuPDF` (`fitz`)
- `pytesseract`
- `Pillow`
- Tesseract OCR 引擎（系统级）

提取后的文本保存为 Markdown 文件，存放于工作目录的 `extracted/` 子目录中。

## Step 2 — 并行深度精读

对每个提取出的 Markdown 文件，启动一个 `coder` 子代理执行 `deep-reader` 分析。

### 子代理提示模板

每个子代理必须：
1. 读取指定的提取文本文件
2. 按 **10 个分析维度** 输出结构化精读报告
3. 保存报告到 `reports/{序号}.report.md`

### 10 个分析维度（必须全部覆盖）

1. **元信息** — 标题、作者、出处、类型、核心主题、写作目的
2. **核心论点** — 中心论点、分论点、逻辑关系
3. **关键概念与定义** — 所有术语，每个附**通俗解释**（比喻/类比）
4. **论据与证据** — 数据、案例、引用、修辞，标注支撑论点
5. **逻辑结构与推理链** — 总体结构、段间衔接、关键推理步骤
6. **方法论** — 研究/分析框架、工具/流程、优势与局限
7. **隐含知识与前提假设** — 默认背景、未经证明的前提、学术背景
8. **重要引用原文** — 3-10句核心原文，标注重要性
9. **知识网络与关联** — 外部领域关联、可延伸主题、可联系理论
10. **批判性思考** — 论证薄弱环节、潜在反驳角度、知识边界

### 注意事项
- 原文若为OCR提取，存在乱码/错误，需根据上下文合理推断修正
- 不得因知识点"不重要"而跳过
- 关键概念必须附带通俗解释
- 输出必须完整，包含末尾自检清单（6项）

## Step 3 — 知识整合

所有子代理完成后，启动一个**整合子代理**（`plan` 或 `coder` 类型），读取所有 `reports/*.report.md`，输出整合总报告。

### 整合报告结构

1. **总概述** — 文件列表、整体定位、核心主题归纳
2. **各文件核心知识点速览** — 每文件：核心论点、关键概念、行动要点
3. **跨文件知识网络** — 反复出现的主题、文件间逻辑关系、层级结构
4. **关键概念词典** — 按主题分类汇总所有概念，简要解释
5. **批判性总结** — 优点、局限性、适用/不适用场景
6. **（可选）行动框架总图** — 若文件含方法论，整合为可执行路线图

### 整合原则
- 不是简单堆砌，要有提炼、层级、逻辑递进
- 保留中文输出
- 结构清晰、层次分明

## 输出规范

### 工作目录约定

- 当用户提供**文件夹路径**时，`{workspace}` 默认为该源文件夹本身，所有中间产物与最终报告直接输出到源文件所在位置，方便与原文件一起管理。
- 当用户提供**多个独立文件**时，`{workspace}` 优先使用用户指定的输出目录；若未指定，则使用第一个文件所在目录。
- 最终报告路径可被用户显式覆盖。

### 中间文件

| 阶段 | 输出 | 路径 |
|------|------|------|
| 文本提取 | 提取后的 Markdown 文件 | `{workspace}/extracted/*.md` |
| 深度精读 | 单文件精读报告 | `{workspace}/reports/{序号}.report.md` |

### 最终交付

| 类型 | 输出路径 |
|------|----------|
| 整合总报告 | 用户指定路径，或 `{workspace}/整合知识点.md` |

## 完整示例

用户请求：

> "帮我批量处理这个文件夹里的所有PDF，提取知识点并整合成一份报告"

执行流程：

1. 列出文件夹中的 `.pdf` / `.srt` / `.md` 等目标文件
2. 将 `{workspace}` 设为该源文件夹
3. 运行 `python3 scripts/extract_pdf_text.py {folder} {workspace}/extracted`
4. 如有 `.srt`，运行 `python3 scripts/srt_to_md.py {folder} {workspace}/extracted`
5. 为每个提取出的 `.md` 文件启动一个并行子代理
6. 等待所有子代理完成
7. 启动整合子代理，读取所有 `reports/*.report.md`
8. 输出整合总报告到 `{workspace}/整合知识点.md` 或用户指定路径

## 注意事项

- **PDF 提取前**，先检查是否已有对应 `.md` 文件，避免重复提取
- **子代理数量** — 若文件超过 10 个，可考虑分批并行（每批 5-10 个）
- **OCR 质量** — 图片型PDF的OCR可能存在错误，子代理需在报告中标注修正
- **语言一致性** — 整合报告的语言与用户输入语言一致
- **不处理** — 加密PDF、扫描质量极差的PDF、非文本文件（如图片、音频、视频），以及无法解析的私有格式
