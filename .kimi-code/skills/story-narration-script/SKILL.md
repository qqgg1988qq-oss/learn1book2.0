# 剧情口播文案生成器

将用户提供的剧情类文章/小说/叙事文案，转化为可直接口播录制的剧情讲解稿。

## 工作流概览

整个流程分为 5 个阶段：

1. **原文解析** — 深度阅读，提取情节要素
2. **情节拆解** — 构建结构化大纲
3. **知识补全** — 搜集背景资料与概念解释
4. **文案撰写** — 按口播规范写出完整讲解稿
5. **自检优化** — 检查口语化、节奏、悬念、字数

## Stage 1: 原文解析

仔细阅读用户提供的原文，提取以下信息：

- **故事类型**：悬疑/爱情/历史/科幻/现实主义/奇幻/其他
- **核心冲突**：主角面临的最大矛盾是什么
- **时间线**：故事跨越的时间段，关键时间点
- **主要角色**：2-5 个核心人物及其关系
- **关键场景**：3-7 个决定剧情走向的重要场景
- **叙事结构**：顺叙/倒叙/插叙/多线并行
- **主题内核**：故事真正想表达的是什么
- **难懂元素**：涉及的专业术语、历史事件、文化背景、地域特色等

**输出**：一份结构化的「原文解析笔记」，保存到工作目录的 `notes-analysis.md`。

## Stage 2: 情节拆解

基于解析笔记，构建三层情节大纲：

### 2.1 主线骨架（一句话概括每段核心事件）

将故事切成 5-10 个「情节块」，每块用一句话概括。例如：
- 块1：男主意外发现妻子隐瞒了惊天秘密
- 块2：男主暗中调查，发现妻子与一个神秘组织有关
- 块3：...

### 2.2 情绪曲线

标注每个情节块的情绪基调：
- 紧张/轻松/悲伤/兴奋/疑惑/温暖/恐惧
- 画出整体情绪起伏，确保有「波浪感」而非平铺直叙

### 2.3 悬念节点

找出 3-5 个最适合制造悬念的转折点，这些将是文案中「留白」或「反问」的位置。

**输出**：`notes-outline.md`，包含主线骨架+情绪曲线+悬念节点。

## Stage 3: 知识补全

针对 Stage 1 中标记的「难懂元素」，进行外部搜索补全：

### 搜索优先级

1. **历史背景**：涉及真实历史事件、时代背景的，搜索相关史料
2. **专业概念**：涉及法律/医学/科技/军事等专业领域的，搜索通俗解释
3. **文化元素**：涉及特定地域文化、民俗、宗教的，搜索文化背景
4. **人物原型**：如有真实人物原型，搜索相关生平
5. **地理信息**：涉及特定地点的，搜索该地点与故事的关联

### 素材整理

将搜集到的素材按「用途」分类整理：
- 「融入文案」：可直接用于讲解中增加深度的素材
- 「参考备用」：帮助你自己理解但不直接写入文案的素材
- 「纠正原文」：发现原文有史实错误或理解偏差的地方

**输出**：`notes-research.md`，分类列出所有搜集到的素材及来源。

## Stage 4: 文案撰写

### 4.1 结构设计

**标准三段式**（3-15 分钟口播适用）：

```
强钩子（3-5秒）→ 背景铺垫（20-30%）→ 主线推进（50-60%）→ 高潮揭示（15-20%）→ 收尾引导（3-5秒）
```

| 段落 | 功能 | 字数占比 | 关键技巧 |
|------|------|---------|---------|
| 开头钩子 | 留住观众 | 3-5% | 悬念/冲突/反常识/直击灵魂的问题 |
| 背景铺垫 | 建立语境 | 15-20% | 时间+地点+主角+核心矛盾，不拖沓 |
| 主线推进 | 展开剧情 | 45-55% | 按情节块推进，每块留一个小悬念 |
| 高潮/转折 | 引爆情绪 | 15-20% | 核心反转+情感爆发点 |
| 结尾收束 | 引导互动 | 5-10% | 金句总结+提问/引导观看原片 |

### 4.2 语言规范

**必须遵守的口播规范**：

- **一句一段**：每句不超过 25 字，超长句必须拆分
- **口语化**：用"咱们""说白了""结果呢""谁能想到"等连接词
- **去文言化**：把"彼时""翌日""未曾想"改为"那时候""第二天""谁也没想到"
- **声音可视化**：在括号内标注语气变化，如（压低声音）（语气加快）（停顿）
- **人名简化**：复杂外国人名用特征代称（如"那个红头发的女探员"），但首次出现给全名
- **数字具象化**："三百公里"改为"从北京开到石家庄那么远"
- **慎用专业术语**：必须使用时，跟一句通俗解释

### 4.3 悬念处理

**三种悬念手法**（至少使用两种）：

1. **信息差式**：「当时所有人都以为事情结束了。但他们不知道的是，真正的噩梦才刚刚开始。」
2. **反问式**：「如果你是他，你会怎么做？选A，继续装傻？还是选B，直接对峙？」
3. **闪前式**：「三天后，当警察破门而入的时候，房间里只剩下一滩血迹和一张字条。」

### 4.4 融入研究素材

将 Stage 3 搜集的素材自然融入文案：
- **历史背景**：在铺垫段融入，增加厚重感（如"这事发生在1987年的深圳，那时候一张火车票就能改变命运..."）
- **专业解释**：在剧情推进中遇到时，用括号旁白补充（如"这种病症在医学上叫'解离性失忆'，说白了就是大脑把太痛苦的记忆锁起来了"）
- **文化注解**：一句话带过，不喧宾夺主（如"按照当地苗族的习俗，这代表着..."）

**输出**：完整的口播文案，保存为 `narration-script.md`。

## Stage 5: 自检优化

完成文案后，逐条检查：

### 5.1 硬性检查

- [ ] 总字数在目标时长范围内（1分钟≈250字，5分钟≈1200字，10分钟≈2500字）
- [ ] 每段不超过 4 行，每行不超过 25 字
- [ ] 没有连续三段都是纯叙述，至少有 1 处互动/反问
- [ ] 没有出现"让我们来看看""接下来""然后然后"等口播废话
- [ ] 所有专业术语都有通俗解释
- [ ] 人名首次出现给全名，之后可用简称/代称

### 5.2 软性检查

- [ ] 开头 3 秒有没有「钩子」？划走率高的平台（抖音/快手）尤其关键
- [ ] 情绪是否有起伏？不是平铺直叙
- [ ] 悬念设置是否过度剧透？保留至少 1 个关键反转让观众看原片
- [ ] 结尾是否有「记忆点」？金句或反问让人记住
- [ ] 整体是否像「朋友聊天」而不是「课文朗读」

### 5.3 修改迭代

如果检查不通过，回到 Stage 4 修改对应段落，再次自检直到全部通过。

最终交付物为一个 Markdown 文件，结构如下：

```markdown

> 预计时长：X分钟 | 字数：XXXX字 | 风格：悬疑/温情/热血/...

---

【开头钩子】
...

【背景铺垫】
...

【主线推进】
...

【高潮转折】
...

【结尾收束】
...
```

## 风格适配

根据故事类型自动调整语气：

| 类型 | 语气特点 | 常用句式 |
|------|---------|---------|
| 悬疑/惊悚 | 压低、神秘、节奏渐快 | "但奇怪的是...""更诡异的是..." |
| 爱情/温情 | 柔和、共情、有停顿 | "说实话...""那一刻..." |
| 历史/史诗 | 沉稳、厚重、有距离感 | "那是XXXX年...""很多人不知道..." |
| 科幻/奇幻 | 惊奇、想象力、可视化 | "想象一下...""如果告诉你..." |
| 现实/社会 | 犀利、直击痛点、有态度 | "说白了...""问题的根源是..." |
