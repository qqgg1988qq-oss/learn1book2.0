# Article to Rich Post — 公众号/头条号富文本排版

## 工作流程

将文章排版为富文本HTML涉及以下步骤：

1. 接收用户输入（文章 + 图片 + 风格偏好 + 目标平台）
2. 分析文章结构（标题层级、重点段落、引用、列表、代码块等）
3. 引导用户选择排版风格（若未指定）
4. 加载对应风格的样式配置（`references/style-presets.md`）
5. 构建富文本HTML（严格遵循 `references/platform-specs.md` 的平台兼容性要求）
6. 生成预览文件并输出

## 第1步：接收用户输入

需要收集的信息：

- **文章内容**：Markdown格式或纯文本（用户粘贴或上传文件）
- **配图**：用户上传的图片文件（可选）
- **视觉场景文件**：`visual-scenes.json`（可选，见下方「图片标注模式」）
- **目标平台**：微信公众号 / 今日头条号 / 双平台（默认双平台）
- **排版风格**：见下方「风格体系」

如果用户没有指定风格，展示风格选项卡片供用户选择。

### 图片标注模式（Visual Scene Annotation）

当用户提供了 `visual-scenes.json` 文件时，启用图片标注模式：

- **触发条件**：用户说"根据 visual-scenes.json 标注"、"在对应位置插入图片标注"、"用文字站位图片"等
- **输入格式**：`visual-scenes.json` 包含 `meta`（文章元信息）和 `prompts` 数组（每个场景有 `scene_id`、`title`、`prompt`）
- **标注方式**：在文章与场景内容对应的段落位置，插入居中灰色斜体文字，格式为 `[图片X：场景标题]`
- **不生成图片展位**：仅插入文字标注，不创建 `<img>` 标签或专门的图片容器
- **标注样式**：使用次要文字色，居中对齐，斜体，与正文段落保持适当间距

**标注插入位置规则（严格遵循）：**
1. **必须每张图都用上**：读取 `visual-scenes.json` 中的所有场景，确保每个 `scene_id` 都有对应的 `[图片X：场景标题]` 文字标注输出，不允许遗漏任何一张图
2. **依照 JSON 设计位置排版**：`visual-scenes.json` 中的场景顺序和标题是根据原文案段落设计的，标注必须插入到与场景标题语义对应的段落之后。优先使用 `scene_id` 顺序和 `title` 关键词定位，而不是重新为图片寻找"最佳位置"
3. 在对应段落的**之后**插入文字标注
4. 如果场景描述与文章某部分内容强相关（如数据、证据、结论），标注插入在该部分内容结束后
5. 标注按 `scene_id` 顺序排列，确保序号连续
6. **图片不能连续出现**：两个图片标注之间必须至少间隔一个正文段落。如果同一位置需要插入多个图片标注，必须在它们之间插入过渡文字、小标题或分割线，确保视觉上不连续堆叠

**示例标注输出：**
```html
<p style="font-size: 14px; line-height: 2.0; color: #a39e99; margin: 25px 0; text-align: center; font-style: italic;">
  [图片3：地球正常的样子——南北极鸟语花香]
</p>
```

## 第2步：分析文章结构

使用LLM能力自动识别文章结构元素：

| 元素 | 识别方式 | 处理方式 |
|---|---|---|
| 主标题 | H1 或首行大字 | 使用标题样式 + 装饰线 |
| 副标题 | H2/H3 或引导语 | 使用副标题样式 |
| 正文段落 | `<p>` 或连续文本 | 使用正文样式，段前段后留白 |
| 引用块 | `>` 或 blockquote | 使用引用框样式（色块+左边框） |
| 重点文字 | `**bold**` 或 关键词 | 使用高亮色+加粗 |
| 无序列表 | `-` 或 `*` | 使用自定义列表样式（圆点/图标） |
| 有序列表 | `1.` `2.` | 使用数字列表样式 |
| 分割线 | `---` 或 空段落 | 使用风格化分割线 |
| 图片 | `![alt](url)` 或 上传文件 | 添加圆角、阴影、居中 |
| 代码块 | 三个反引号 | 使用暗色代码块样式 |
| 链接 | `[text](url)` | 使用品牌色下划线 |

## 第3步：风格体系

提供 **8种预设风格**，每种风格包含完整的CSS样式配置、配色方案、字体搭配和装饰元素。

**风格速览表：**

| 风格 | 适用场景 | 配色基调 | 字体气质 |
|---|---|---|---|
| 1. 现代简约 | 通用/职场/观点 | 黑白灰+单强调色 | 清晰专业 |
| 2. 时尚杂志 | 潮流/生活方式 | 高对比+大胆色块 | 前卫有型 |
| 3. 文艺清新 | 文学/情感/美学 | 低饱和暖色系 | 温暖治愈 |
| 4. 高端商务 | 企业/财经/品牌 | 深蓝/深灰+金 | 沉稳大气 |
| 5. 科技极客 | 互联网/产品/科技 | 深色底+荧光色 | 简洁高效 |
| 6. 干货科普 | 知识/教育/方法 | 白底+活泼强调色 | 逻辑清晰 |
| 7. 东方美学 | 文化/历史/传统 | 中国红/墨/金 | 典雅古朴 |
| 8. 条漫叙事 | 故事/幽默/GQ式 | 纯黑文字+极简 | 节奏感强 |

每种风格的详细CSS配置见 `references/style-presets.md`。

> 用户说"帮我选个风格"时：
> - 职场/商业/财经内容 → 推荐现代简约或高端商务
> - 文学/情感/生活 → 推荐文艺清新
> - 科技/互联网/产品 → 推荐科技极客
> - 潮流/时尚/美妆 → 推荐时尚杂志
> - 教育/知识/方法 → 推荐干货科普
> - 文化/历史/传统 → 推荐东方美学
> - 轻松/幽默/故事 → 推荐条漫叙事

## 第4步：加载样式配置

根据用户选择，从 `references/style-presets.md` 中提取对应的CSS变量和HTML模板。

**通用规则（所有风格共享）：**

- 正文字号：15px（手机阅读最佳）
- 行高：1.75（阅读舒适度最优）
- 段落间距：20px
- 图片最大宽度：100%，自动居中
- 图片圆角：6px（大多数风格）
- 引用块左边框：3px实心线
- 代码块背景：#f6f6f6
- 所有颜色使用具体色值，不使用CSS变量（兼容公众号编辑器）

## 第5步：构建富文本HTML

### 公众号HTML兼容性要求

微信公众号编辑器对HTML有严格限制，必须遵循以下规则：

**支持的标签（白名单）：**
```
<section>, <p>, <span>, <img>, <strong>, <em>, <a>, <br>,
<h1>-<h6>, <ul>, <ol>, <li>, <blockquote>, <pre>, <code>
```

**CSS支持：**
- 仅支持行内 style 属性（`style="..."`）
- 不支持外部CSS文件
- 不支持 class 选择器（需内联所有样式）
- 部分CSS属性被过滤，安全的属性包括：
  - `color`, `background-color`, `font-size`, `font-weight`
  - `line-height`, `text-align`, `padding`, `margin`
  - `border`, `border-left`, `border-radius`
  - `width`, `max-width`, `height`, `display`

**图片处理：**
- 图片必须上传到微信服务器获得 `data-src` 属性
- 本地图片先用 `<img src="本地路径">` 占位，用户替换为微信图床链接
- 封面图建议尺寸：900×383px
- 正文图片宽度：≤1080px

**输出格式示例：**

```html
<section style="margin: 20px 0; padding: 0 16px;">
  <h2 style="font-size: 20px; font-weight: bold; color: #333; margin: 30px 0 15px; border-bottom: 2px solid #e0e0e0; padding-bottom: 10px;">
    这里是标题
  </h2>
  <p style="font-size: 15px; line-height: 1.75; color: #3f3f3f; margin: 20px 0; text-align: justify;">
    这里是正文内容，<strong style="color: #1a1a1a;">重点文字</strong>使用加粗和深色。
  </p>
  <blockquote style="margin: 20px 0; padding: 15px 20px; background: #f8f9fa; border-left: 3px solid #1890ff; color: #666; font-size: 14px; line-height: 1.7;">
    这里是引用内容
  </blockquote>
  <p style="text-align: center; margin: 25px 0;">
    <img src="image.jpg" style="max-width: 100%; border-radius: 6px; display: inline-block;">
  </p>
</section>
```

### 头条号HTML适配

头条号编辑器支持的标签与公众号类似，但差异点：
- 支持更多语义化标签
- 图片处理方式不同（头条号自动处理图片上传）
- 代码块样式更受限
- 标题层级限制更多

生成头条号版本时：
- 简化嵌套结构
- 避免过深的 `<section>` 嵌套
- 图片URL使用占位符，用户发布后替换

## 第6步：输出

生成以下文件保存到输出目录：

| 文件 | 说明 |
|---|---|
| `{name}_wechat.html` | 微信公众号版富文本HTML |
| `{name}_toutiao.html` | 头条号版富文本HTML |
| `{name}_preview.html` | 本地预览版（含完整样式，可浏览器打开预览） |
| `images/` | 用户上传的图片（处理后） |

**图片标注模式输出：**
当启用了图片标注模式（提供了 `visual-scenes.json`）时，上述3个HTML文件中会自动在对应位置插入 `[图片X：场景标题]` 文字标注。用户后续在公众号/头条号编辑器中，只需将文字标注替换为实际图片即可。

**文件头注释**（每个HTML文件顶部）：
```html
<!-- 
  排版风格：{style_name}
  目标平台：{platform}
  生成时间：{datetime}
  使用说明：
    1. 公众号：全选复制 → 粘贴到公众号编辑器
    2. 头条号：全选复制 → 粘贴到头条号编辑器
    3. 本地预览：直接用浏览器打开 preview.html
-->
```

## 质量标准

- HTML结构必须严格使用行内style，无外部CSS依赖
- 只使用微信公众号白名单内的HTML标签
- 配色方案符合WCAG AA对比度标准（文字与背景对比度≥4.5:1）
- 图片在375px-414px宽度手机上显示良好
- 段落行高和间距确保移动端阅读舒适度
- 代码块在深色/浅色模式下均可辨识
- **图片标注模式专项检查**：
  - [ ] `visual-scenes.json` 中的每个场景都生成了对应的 `[图片X：场景标题]` 标注
  - [ ] 标注位置与 JSON 设计时的段落位置一致，未擅自改动
  - [ ] 任意两个图片标注之间至少有一个正文段落、小标题或分割线隔开
  - [ ] 不存在连续堆叠的图片标注

## 附加功能

**自动优化：**
- 过长的段落自动分段（每段≤200字为最佳）
- 连续3个以上列表项自动添加间隔色
- 检测到时间/日期/数据自动使用等宽字体
- 文章开头自动添加引导阅读语（可选）
- 文章结尾自动添加关注引导/分隔线（可选）

**图片处理：**
- 自动居中并添加适当边距
- 图片下方支持添加图注（检测alt文本）
- 多张连续图片自动添加间距
- 支持图片圆角、轻微阴影效果

**图片标注模式（Visual Scene Annotation）：**
- 根据 `visual-scenes.json` 中的场景信息，在文章对应位置自动插入 `[图片X：场景标题]` 文字标注
- 标注样式：次要文字色、居中对齐、斜体，不占用图片展位
- 支持场景与文章内容的语义匹配，自动找到最佳插入位置
- 标注按 `scene_id` 顺序排列，确保序号连续
- 用户后续只需将文字标注替换为实际图片即可
