# Article to Visual Rich Post — 文章视觉化+图文排版一站式工作流

将任意文章自动完成 **视觉场景拆解 → 图文混排 HTML 生成**，最终输出两个文件：

1. `visual-scenes.json` — 静态视觉场景描述文件
2. `article.html` — 已排好版面的富文本 HTML

> **核心原则**：
> - `visual-scenes.json` 中的每个场景都必须被 HTML 使用
> - 场景占位符位置必须严格依照 JSON 设计时的段落对应关系
> - 任意两个场景占位符之间必须有文字段落隔开，禁止连续堆叠
> - 不区分公众号/头条号，只输出一个通用 HTML

---

## 输出文件

```
{output-dir}/
├── visual-scenes.json    # 视觉场景描述文件
└── article.html          # 排好版面的富文本 HTML
```

### visual-scenes.json 格式

```json
{
  "meta": {
    "title": "视频/文章标题",
    "source": "源文件路径",
    "scene_count": 20,
    "style_summary": "画面风格概述",
    "global_style_prompt": "全局画面基调描述"
  },
  "prompts": [
    {
      "scene_id": 1,
      "title": "场景标题",
      "keywords": ["可选：用于匹配文章段落的关键词数组"],
      "prompt": "100-200字静态画面描述"
    }
  ]
}
```

### article.html 格式

- 使用 `<section>` 作为最外层容器
- 使用 `<p>`、`<h1>`、`<h2>`、`<blockquote>`、`<hr>` 等安全标签
- 所有 CSS 必须内联到 `style` 属性中
- 场景使用文字占位符：`[ scene_id:场景标题 ]`
- 占位符以居中段落形式插入，不嵌入 `<img>` 标签

## 工作流程

### 步骤 1：接收并分析文章

读取用户提供的文章/文案，分析其结构：

- 主标题、副标题、正文段落
- 引用块、重点文字、列表、分割线
- 开头钩子、主体章节、结尾引导
- 识别每个段落的内容类型（概念解释、故事叙述、历史回顾、引用展示、批判性分析等）

### 步骤 2：确认视觉风格

向用户展示可用的视觉风格预设，等待用户选择：

| 序号 | 风格名称 | 适用场景 |
|------|---------|---------|
| 1 | `warm-isometric` | 知识科普、文化内容、教育视频、社交媒体 |
| 2 | `academic-tech` | 哲学思辨、学术解读、深度内容、思想史回顾 |
| 3 | `literary-warm` | 文学解读、散文、人物故事、温情叙事 |
| 4 | `documentary-grey` | 历史回顾、调查报道、严肃新闻、社会观察 |
| 5 | `hand-drawn-journal` | 轻松知识科普、课堂笔记、手账风格内容 |
| 6 | `lofi-chill-vector` | 现代数字插画、高饱和撞色、Lo-Fi Chill 氛围、哲学/文艺内容 |

用户也可以提供自定义风格描述。

### 步骤 3：生成 visual-scenes.json

基于文章结构和用户选择的视觉风格：

- 将文章拆解为 **15-25 个场景**
- 每个场景覆盖一个完整的讲解单元
- 每个场景的 `title` 必须与文章中对应段落语义相关
- `prompt` 严格遵循选定风格的视觉特征
- 所有画面描述必须是静态的，禁止动画/运动/镜头/特效描述

### 步骤 4：确认排版风格

向用户展示可用的排版风格预设，等待用户选择：

| 风格 | 适用场景 | 配色基调 |
|---|---|---|
| 1. 现代简约 | 通用/职场/观点 | 黑白灰+单强调色 |
| 2. 时尚杂志 | 潮流/生活方式 | 高对比+大胆色块 |
| 3. 文艺清新 | 文学/情感/美学 | 低饱和暖色系 |
| 4. 高端商务 | 企业/财经/品牌 | 深蓝/深灰+金 |
| 5. 科技极客 | 互联网/产品/科技 | 深色底+荧光色 |
| 6. 干货科普 | 知识/教育/方法 | 白底+活泼强调色 |
| 7. 东方美学 | 文化/历史/传统 | 中国红/墨/金 |
| 8. 条漫叙事 | 故事/幽默/GQ式 | 纯黑文字+极简 |

### 步骤 5：生成 article.html

根据 `visual-scenes.json` 和排版风格生成 HTML：

1. **预处理原文**：移除口播/朗读类提示词，包括 `（停顿）`、`（停顿X秒）`、`（放慢语速）`、`（压低声音）`、`（提高语调）`、`（笑）` 等，只保留适合图文阅读的纯文本
2. 分析文章结构，应用排版样式
3. 找到每个场景标题对应的段落位置
4. 在对应段落后插入文字占位符 `[ scene_id:场景标题 ]`
5. 确保每个场景都被使用
6. 确保任意两个占位符之间至少间隔一个正文段落
7. 添加文件头注释

### 步骤 6：输出文件

将 `visual-scenes.json` 和 `article.html` 保存到用户指定目录。

## 视觉风格预设

### `warm-isometric` —— 温暖治愈等距3D插画风

- **氛围**：温馨治愈、知识友好、可爱亲切
- **视觉倾向**：等距透视(2.5D)，扁平化+柔和立体阴影，米色/奶油色/浅棕色暖基调
- **典型元素**：Q版微型人物、微型场景摆件、木牌/标牌、可爱图标

### `academic-tech` —— 学术科技风

- **氛围**：理性、克制、信息密度高
- **视觉倾向**：几何线条和信息图为主，深色底配亮色点缀
- **典型元素**：数据面板、几何图表、符号系统、电子标签

### `literary-warm` —— 文学温暖风

- **氛围**：温暖、柔和、有人文气息
- **视觉倾向**：暖色调为主，复古纸张质感
- **典型元素**：手写笔触、暖光、插画感画面

### `documentary-grey` —— 纪实灰调风

- **氛围**：冷静、客观、有历史厚重感
- **视觉倾向**：黑白灰为主，偶现暖色点缀
- **典型元素**：档案照片、时间轴、文献质感

### `hand-drawn-journal` —— 手绘涂鸦手账风

- **氛围**：轻松亲切、活泼可爱、像课堂笔记
- **视觉倾向**：米黄色笔记本纸张背景、黑色手绘抖动感轮廓线、鲜艳马克笔填充色
- **典型元素**：简笔画火柴人角色（点状眼睛、弧线嘴巴）、星星、爱心、灯泡、云朵、对话框、便利贴、手写体与印刷体混排的中文文字

### `lofi-chill-vector` —— Lo-Fi Chill 撞色矢量插画风

- **氛围**：松弛、复古未来感、适合深夜阅读和哲学沉思的 Lo-Fi Chill 氛围
- **视觉倾向**：现代数字插画，平面矢量质感，深色清晰描边，色块分明无渐变，灰绿色调几何拼贴背景
- **典型配色**：珊瑚粉 `#FF6B6B`、深蓝紫 `#4A2C8A`、薄荷绿 `#4ECDC4`、米黄 `#FFE66D`
- **典型元素**：几何色块拼贴、复古磁带/黑胶/台灯等 Lo-Fi 小物件、扁平化人物、抽象符号与文字标牌

## 视觉场景 prompt 写作规范

### 默认风格要求

`prompt` 的具体写法由用户在步骤 2 中确认的视觉风格决定。所有 prompt 必须：

1. **静态描述**：禁止任何动画、运动、镜头、转场、特效描述
2. **100-200字**：包含背景、主体、构图、色彩、文字元素
3. **风格一致**：严格遵循选定风格的视觉特征
4. **文字载体**：可使用木牌、标牌、竹简、气泡框、电子标签等

### 禁止出现的描述

- **动画类**：动画、动态、运动、移动、滑动、旋转、缩放、推近、拉远、淡入、淡出、闪烁、呼吸、抖动、粒子、消散、汇聚、爆裂、故障、glitch、扫描线、光流、缓缓、逐渐、慢慢、快速、突然
- **镜头类**：镜头、机位、视角、跟随、穿越、穿过、环绕、俯视、仰视、特写、全景、近景、远景、摇镜、移轴、Ken Burns
- **时间类**：然后、接着、之后、随即、约 X 秒后、X 秒后
- **特效类**：特效、光效、粒子效果、波纹、涟漪、3D 旋转、发光脉动

## HTML 构建规范

### 标签白名单

```
<section>, <p>, <span>, <strong>, <em>, <a>, <br>,
<h1>-<h6>, <ul>, <ol>, <li>, <blockquote>, <pre>, <code>, <hr>
```

### CSS 规则

- 仅使用行内 `style` 属性
- 不使用 class 或 id
- 颜色使用具体色值
- 使用系统字体栈

### 场景占位符插入规则（核心）

1. **必须每个场景都用上**：`visual-scenes.json` 中的每个 `scene_id` 都必须在 HTML 中生成对应的文字占位符
2. **依照 JSON 设计位置排版**：占位符必须插入到与场景标题语义对应的段落之后
3. **占位符不连续**：两个占位符之间必须至少间隔一个正文段落、小标题或分割线
4. **占位符格式**：统一使用 `[ scene_id:场景标题 ]`
5. **不使用 `<img>` 标签**：HTML 中不嵌入任何图片，只保留文字占位符，供用户后续手动替换或配图
6. **占位符样式**：使用简洁的文字段落占位，不使用背景框、边框、圆角等装饰性样式，避免视觉喧宾夺主

```html
<p style="text-align: center; color: #2c5aa0; margin: 24px 0;">
  [ 3:旧观点：文化是固定本质 ]
</p>
```

## 质量标准

- [ ] `visual-scenes.json` 包含 15-25 个场景
- [ ] 每个场景的 `prompt` 都是 100-200 字静态描述
- [ ] 所有场景的视觉风格统一
- [ ] HTML 严格使用行内 style，无外部 CSS 依赖
- [ ] 只使用白名单内的 HTML 标签
- [ ] HTML 中不包含任何 `<img>` 标签
- [ ] `visual-scenes.json` 中的每个场景都生成了对应的 `[ scene_id:场景标题 ]` 文字占位符
- [ ] 占位符位置与 JSON 设计时的段落位置一致
- [ ] 任意两个占位符之间至少有一个正文段落、小标题或分割线
- [ ] 不存在连续堆叠的占位符
- [ ] 已移除口播/朗读类提示词（停顿、放慢语速等）
- [ ] 配色方案符合 WCAG AA 对比度标准
