# content-to-script

## 用途

将 `deep-reader` 生成的精读报告（10维度分析）转化为适合视频口播的文案脚本。

## 输入

一份标准的精读报告 Markdown 文件，包含以下维度：
- 元信息（标题、作者、核心主题）
- 核心论点（中心论点 + 分论点）
- 关键概念与定义（含通俗解释）
- 论据与证据
- 逻辑结构
- 方法论
- 隐含知识与前提假设
- 重要引用原文
- 知识网络与关联
- 批判性思考

## 输出

一份结构化的口播文案，包含：
- 时长预估（基于字数）
- 开头钩子（引发好奇的问题/冲突/场景）
- 主体内容（分节展开，每节有过渡句）
- 结尾引导（总结 + 互动/关注引导）
- 语气标注（情绪起伏、停顿、重音提示）

## 使用流程

```
/content-to-script <精读报告文件路径>
```

或直接在对话中粘贴精读报告内容并说"转成口播文案"。

**Step 1 — 询问时长**

技能触发后，首先询问用户需要的视频时长：

| 选项 | 时长 | 适用场景 | 字数范围 |
|------|------|---------|---------|
| 短视频 | 1-2分钟 | 抖音/小红书/视频号快知识 | 300-500字 |
| 中视频 | 5-15分钟 | B站/YouTube标准知识视频 | 1200-3500字 |
| 长视频 | 15-40分钟 | B站深度/播客/讲座 | 3500-10000字 |

**Step 2 — 生成文案**

根据用户选择的时长，自动调整内容取舍策略：
- **短视频**：只保留最核心的1个论点+1个类比+金句结尾
- **中视频**：保留2-3个分论点+2个类比+完整起承转合
- **长视频**：覆盖4-6个分论点+3个以上类比+完整结构+批判性思考

**Step 3 — 输出文件**

文案自动保存为 `.md` 文件，位置与输入文件**同文件夹**，命名为：

```
{原文件名}-script.md
```

例如：
- 输入：`reports/01-弥散的心智-report.md`
- 输出：`reports/01-弥散的心智-report-script.md`

## 转化原则

1. **口语优先**：将学术表述转为日常口语，但保留核心概念
2. **钩子开头**：前30秒必须抓住注意力（反常识、悬念、冲突）
3. **类比驱动**：利用精读报告中的通俗类比，用故事化方式展开
4. **节奏控制**：每200-300字一个"呼吸点"，避免信息密度过高
5. **互动感**：适当加入"你有没有想过""说实话"等口语化互动
6. **分层递进**：从具体到抽象，从故事到理论，再从理论回到生活
7. **避免中英文混用**：口播文案以中文表达为主，不要用英文单词替代中文常用词（例如把"抵触"写成"resistance"），除非该英文词是约定俗成的品牌名、专业术语且无合适中文替代
