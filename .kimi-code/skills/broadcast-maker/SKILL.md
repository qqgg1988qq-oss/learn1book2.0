# Broadcast Maker — 文章到口播文案一站式工作流

将任意文章通过 **深度精读 → 口播转化 → 文件输出** 三步流水线，自动生成可直接录制的视频文案。

## 工作流概览

```
用户输入文章
    ↓
[Step 1] Deep Read — 10维度深度精读分析
    ↓
生成结构化精读报告 → 保存为 {源文件名}-deep-read.md
    ↓
[Step 2] Ask Duration — 询问视频时长
    ↓
[Step 3] Script Write — 转化为口播文案 → 保存为 {源文件名}-script.md
    ↓
两个文件均保存到源文件所在目录（或用户指定的输出目录）
```

## 使用方式

```
/broadcast-maker ./article.md
```

或直接在对话中粘贴文章内容并说"做成视频文案"。

## Step 1 — 深度精读（自动执行）

接收到文章后，立即按照 `deep-reader` 的标准进行10维度分析：

### 分析维度（必须全部覆盖）

1. **元信息** — 标题、作者、出处、类型、核心主题
2. **核心论点** — 中心论点与分论点、逻辑关系
3. **关键概念与定义** — 所有专业术语 + 通俗解释（比喻/类比）
4. **论据与证据** — 数据、案例、引用、修辞，标注论据→论点的支撑关系
5. **逻辑结构与推理链** — 总体结构、段落衔接、关键推理步骤
6. **方法论** — 研究方法、分析框架、优势与局限
7. **隐含知识与前提假设** — 默认背景知识、未经证明的前提、学术背景
8. **重要引用原文** — 3-10句核心原文 + 重要性标注
9. **知识网络与关联** — 外部领域关联、可延伸主题、理论概念
10. **批判性思考** — 论证薄弱环节、潜在反驳角度、知识边界

### 精读原则

- 不得因知识点"不重要"而跳过
- 多层次解析：字面 → 深层逻辑
- 关键概念必须附带通俗解释（比喻/类比）
- 输出必须完整，包含自检清单

## Step 2 — 询问时长（自动执行）

精读完成后，向用户确认视频时长：

| 选项 | 时长 | 适用场景 | 字数范围 |
|------|------|---------|---------|
| 短视频 | 1-2分钟 | 抖音/小红书/视频号快知识 | 300-500字 |
| 中视频 | 5-15分钟 | B站/YouTube标准知识视频 | 1200-3500字 |
| 长视频 | 15-40分钟 | B站深度/播客/讲座 | 3500-10000字 |

**默认**：若用户未明确选择，推荐中视频（5-15分钟）。

## Step 3 — 口播文案创作（自动执行）

根据精读报告和用户选择的时长，按照 `content-to-script` 的标准生成文案。

### 时长对应的内容策略（APAG 四段式）

- **短视频**：第一段钩子 + 第三段中 1 个核心优势/论点 + 1 个类比 + 第四段行动号召
- **中视频**：完整 APAG 四段，第二段 2-3 个错误信念/痛苦，第三段 2-4 个分论点/前后对比 + 2-3 个类比，第四段 3 个行动步骤
- **长视频**：完整 APAG 四段，第二段深入 3-4 个错误信念/失败模式，第三段覆盖 4-6 个分论点 + 3 个以上类比 + 批判性思考，第四段 4-5 个行动步骤

### 文案结构：APAG 四段式

口播文案必须按 **APAG 说服框架**组织，不采用传统的「开头-主体-升华-结尾」结构。APAG 四段依次为：

1. **Attention（钩子 / 标题）** — 30秒内制造信息鸿沟，抓住注意力
2. **Perspective（视角 / 问题）** — 描绘观众当前的错误信念、痛苦与失败模式，让观众感到被理解
3. **Advantage（优势 / 解决方案）** — 提出更优视角、核心概念、证据与前后对比，让观众相信存在更好的路
4. **Amplify/Gamify（放大 / 游戏化行动阶梯）** — 把转变拆解为 3-5 个可执行步骤，给出明确 CTA

各段要求详见下文。

---

### Attention（钩子 / 标题）

**功能**：制造「信息鸿沟」，让观众产生"我必须看下去"的好奇心。

**常用元素**：反直觉观点、问题、数字、时间框架、具体场景、痛点共鸣。

**评估标准**：
- **相关性**：与目标观众的日常生活有多相关？
- **认知水平**：复杂度是否匹配观众的理解水平？
- **获取成本**：观众多快能从视频中获得价值？

**关键原则**：先写正文，最后从正文中提炼最佳钩子。不要凭空编造一个与正文脱节的标题党。

**开头钩子模板库（每次随机选择一种）**

**禁止**每次都用同一种开头。根据文章特点，从以下模板类型中**随机选择一种**，或灵活组合：

| 类型 | 核心策略 | 示例句式（全部用中文表达） |
|------|----------|----------|
| **反常识冲击型** | 直接否定普遍认知，制造认知冲突 | "大部分人以为……但其实……" / "我们从小听到的……可能从一开始就被误解了" |
| **惊人数据型** | 开篇抛出反直觉数字，用数据破题 | "你知道……的误差只有……吗？" / "……年前的人，已经能做到……" |
| **场景代入型** | 把观众拉进一个具体情境 | "想象一下，你穿越回……年，面前摆着……" / "如果把你放到……，你该怎么……" |
| **故事引入型** | 从一个轶事、案例或人物切入 | "……年前，有一个人做了一件……的事" / "有一件文物，藏着……的秘密" |
| **金句引用型** | 用原文金句或名言破题 | "《……》里有一句话：……。这句话背后，藏着……" / "古人曾说……，但很少有人知道，这句话的真正含义是……" |
| **挑战常识型** | 直接断言大众认知错误 | "关于……，你学的可能是错的" / "教科书没告诉你的是……" |
| **时间对比型** | 古今并置制造张力 | "我们今天用……就能……，但……年前的人，用……做到了同样的事" |
| **个人共鸣型** | 以"我也曾……"拉近距离 | "说实话，我以前也觉得……，直到我读了……" / "如果你和我一样，曾经认为……，那今天这期视频可能要刷新你的认知" |
| **物件聚焦型** | 从一个具体物品展开 | "你面前的这个……，其实隐藏着……" / "别小看……，它可能是人类历史上最重要的……" |
| **悬念钩子型** | 抛出一个不解释的结论 | "接下来我要说的这件事，学界争论了……年" / "有一件事，古人做到了，但现代科学还解释不了" |

**选择原则**：
- 文章有惊人数据 → 优先用**惊人数据型**或**时间对比型**
- 文章颠覆传统认知 → 优先用**反常识冲击型**或**挑战常识型**
- 文章有核心人物/文物 → 优先用**故事引入型**或**物件聚焦型**
- 文章有金句原文 → 优先用**金句引用型**
- 内容较抽象 → 用**场景代入型**降低门槛
- 想快速拉近观众 → 用**个人共鸣型**

---

### Perspective（视角 / 问题）

**核心**：描绘观众当前的问题图景，让观众感到"这说的就是我"。

**不要立刻给答案**。先展示：
- 观众当前的**错误信念**是什么？
- 这个信念带来了什么**痛苦**？
- 常见的**失败模式**有哪些？
- 用一个**具体场景**把困境写活

**目标**：让观众在镜子前看清自己「衣服穿反了」。

**内容来源**：从精读报告中提取的核心论点、隐含前提、批判性思考中，找出与观众现实最冲突的部分。

---

### Advantage（优势 / 解决方案）

**核心**：销售一个更好的视角或做事方式。

**必须包含**：
- **新颖观点 / 核心概念**：用一句话提出更优视角
- **之前-之后对比**：旧视角 vs 新视角，至少 3 组对比
- **社会证明 / 权威引用**：原文数据、案例、专家引用、作者经验
- **类比 / 隐喻**：用精读报告中的通俗类比，把抽象概念讲清楚

**禁止写成报告式提纲**：
- 口播正文里**不得出现** "核心新观点"、"之前-之后对比"、"支撑证据"、"关键概念"、"权威引用"、"原文作者经验" 这类报告小标题。
- 不要把优势段切成 "先抛观点 → 再列对比表 → 再列证据 bullet → 再解释概念" 的四大块。
- 所有元素必须用**连续的口语段落**自然编织：一个观点讲完后，用 "你可能会问"、"我举个例子"、"反过来想"、"这不是凭空说的" 等口语过渡，把对比、证据、概念串进同一段叙述里。
- 表格只可用于视频制作备注或字幕提示，**不能直接放在口播正文**中朗读。

**目标**：让观众相信"存在一条更近、风景更好的路"，同时感觉像是一个真实的人在镜头前讲故事，而不是在念论文提纲。

---

### Amplify/Gamify（放大 / 游戏化行动阶梯）

**核心**：把认知转变变成可执行的阶梯。

**必须包含**：
- **3-5 个具体步骤**：每个步骤有清晰动作
- **每个步骤的即时收益**：让观众知道做这一步有什么好处
- **降低行动摩擦的小技巧**：降低启动难度
- **明确的 CTA**：视频结尾明确告诉观众下一步做什么

**目标**：像游戏任务树一样，让观众从"知道"走到"做到"。

**时长对应的内容策略**：

- **短视频**：只保留第一段钩子 + 第三段中 1 个核心论点与类比 + 第四段行动号召
- **中视频**：完整四段，保留 2-4 个分论点 / 前后对比 + 2-3 个类比 + 3 个行动步骤
- **长视频**：完整四段，覆盖 4-6 个分论点 + 3 个以上类比 + 4-5 个行动步骤 + 批判性思考

**输出标签规范**：
- 最终口播文案的小标题、段落标签必须使用中文，不得出现 "Attention"、"Perspective"、"Advantage"、"Amplify/Gamify"、"CTA" 等英文标签。
- 允许在文件顶部注释中说明"本文案遵循 APAG 四段框架"，但口播正文及各级标题必须为中文。

### 转化原则

1. **口语优先**：学术表述 → 日常口语，保留核心概念
2. **类比驱动**：用精读报告中的通俗类比，故事化展开
3. **节奏控制**：每200-300字一个"呼吸点"（停顿、语气变化）
4. **互动感**：加入"你有没有想过""说实话""你发现了吗"等口语化互动
5. **分层递进**：具体 → 抽象 → 理论 → 回到生活
6. **叙事优先于结构**：把报告式结构转化为连续讲述。尤其第三段（Advantage）要把观点、对比、证据、概念编织成一段有起承转合的口语叙事，避免 "小标题 + bullet" 的大纲形态。
7. **严禁中英文混用**：
   - 口播文案必须使用**纯中文表达**，禁止在中文句子中夹杂英文单词、缩写或短语。
   - 例如：禁止使用 "routine"、"hook"、"CTA"、"skill"、"flow"、"entropy"、"identity"、"mindset"、"focus" 等英文词；必须替换为对应中文：日常安排、钩子、行动号召、技能、心流、熵增/混乱、身份认同、思维模式、专注。
   - 唯一例外：约定俗成的品牌名（如 耐克、苹果）、人名、专有学术名词在首次出现时可用中文译名加括号备注，但口播正文仍读中文。
   - 所有标题、小标题、标注、示例句式也必须使用中文。
   - 如果原文是英文，引用时需先翻译成通顺中文，再用"作者原文大意是……"的方式表达，不要直接插入英文原句。
   - 生成完成后必须自检：通读全文，若发现任何中英混用，立即替换为纯中文。

### 语气标注

文案中需包含以下标注：

| 标记 | 含义 |
|------|------|
| （停顿） | 1-2秒沉默，制造节奏感 |
| （停顿0.5秒） | 短停顿，逗号级 |
| （笑） | 自然轻笑，增加亲和力 |
| （放慢语速） | 降低语速，增强氛围感 |
| （压低声音） | 降低音量，制造紧张/神秘感 |
| （提高语调） | 提高音调，表达意外/兴奋 |

## 文件输出

### 输出路径约定

- **默认**：当用户输入的是文件路径时，口播文案保存到**源文件所在目录**，文件名：`{源文件名}-script.md`。
- **备用**：若用户粘贴的是纯文本，或明确要求保存到 Obsidian 博客目录，则保存到 `/Users/chouchou/Documents/Obsidian Vault/成长计划/博客/{文件名}-script.md`。
- **用户指定**：若用户给出了具体输出目录，优先使用用户指定路径。

### 输出文件清单

每次执行会输出 **两个文件**：

| 文件 | 命名规则 | 内容 |
|------|---------|------|
| **精读报告** | `{源文件名}-deep-read.md` | 完整的10维度深度精读分析 |
| **口播文案** | `{源文件名}-script.md` | 结构化口播文案 + 制作建议 |

### 文件名规则

- 若用户输入的是文件路径，取原文件名（去掉扩展名）+ `-deep-read.md` / `-script.md`
- 若用户粘贴的是纯文本，取文章标题（或前10个字的拼音/英文）+ `-deep-read.md` / `-script.md`

### 精读报告文件内容

输出的 `-deep-read.md` 包含：
1. 元信息（标题、作者、出处、类型、核心主题）
2. 核心论点与分论点
3. 关键概念与定义（含通俗类比）
4. 论据与证据
5. 逻辑结构与推理链
6. 方法论分析
7. 隐含知识与前提假设
8. 重要引用原文
9. 知识网络与关联
10. 批判性思考
11. 自检清单

### 口播文案文件内容

输出的 `-script.md` 包含：
1. 文章元信息（标题、作者、来源）
2. 视频时长与字数预估
3. APAG 四段式完整口播文案（含语气标注）
   - Attention（钩子 / 标题）
   - Perspective（视角 / 问题）
   - Advantage（优势 / 解决方案）
   - Amplify/Gamify（行动阶梯 / 行动号召）
4. 视频制作建议（可选画面素材、字幕重点、BGM建议）
5. 自检清单（重点检查是否有中英文混用、APAG 四段是否完整、是否存在报告式提纲结构）

## 注意事项

- 精读报告会单独保存为 `{源文件名}-deep-read.md`，与口播文案放在同一输出目录
- 口播文案的最终输出默认保存到源文件所在目录；只有用户明确要求或源目录不可写时，才保存到 Obsidian 博客目录
- 若用户要求调整（增删故事、改变语气、调整时长），直接修改文案并重新保存
- 保留 deep-reader 和 content-to-script 各自的核心能力，不简化任何分析维度
