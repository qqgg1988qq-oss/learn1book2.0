# OCR Corrector 子代理任务模板

## 任务信息

- **文件路径**: `{chapter-file-path}`
- **章节标题**: `{chapter-title}`
- **预计修复类型**: [乱码/断裂/空行/页码/错字]

## 校正检查清单

### 1. 乱码片段
- [ ] 删除孤立的无意义英文单词（如 `GB wenwor`、`a7`）
- [ ] 删除无意义的数字+字符组合（如 `40 一`）
- [ ] 删除纯符号行（如 `■ □ ○` 单独一行）

### 2. 断裂句子
- [ ] 检查行尾是否有未完成的句子被截断
- [ ] 将错误换行合并为连续段落
- [ ] 保留正确的段落分隔（空行分隔的段落保持）

### 3. 多余空行
- [ ] 检查连续空行数量
- [ ] 将3个以上连续空行合并为最多2个

### 4. 页码残留
- [ ] 删除形如 `— 123 —` 的页码
- [ ] 删除形如 `40 一` 的页码
- [ ] 删除形如 `第 45 页` 的页码

### 5. OCR 错字
- [ ] 检查上下文不通顺的词语
- [ ] 修复明显的形近字误识
- [ ] **跳过专业术语**（即使不认识）

## 修复原则

1. **只修 OCR 错误**：不做内容改写、不增删原文意思
2. **保留格式**：YAML frontmatter、标题、列表、引用保持原样
3. **专业术语不动**：哲学概念、人名、地名保持原样
4. **不确定的不改**：宁可少改不错改
5. **直接修改文件**：用 Edit 工具修改原文件，不生成新文件

## 输出要求

完成后汇报：
- 修复了多少处
- 主要修复了哪些类型的问题
- 举一个最典型的修复示例（修复前 vs 修复后）
