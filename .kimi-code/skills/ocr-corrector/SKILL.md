# OCR Corrector — OCR 文本校正器

自动检测并修复扫描 PDF 经 OCR 转换后的 Markdown 文件中的排版错误。

## 使用方式

```
/ocr-corrector /Users/xxx/learnAbook/有意识的心灵：一种基础理论研究
```

## 工作流程

### Step 1: 验证输入路径

- 路径必须是**绝对路径**
- 路径下必须存在 `chapters/` 目录
- 可选：`chapters/_index.md` 提供章节顺序信息

### Step 2: 扫描待校正文件

列出 `chapters/` 下所有 `.md` 文件，排除 `_index.md` 和非内容章节（如 `front-*-目录.md`、`back-*-参考文献.md`）。

### Step 3: 并行校正（核心）

对每个内容章节文件，**并行启动子代理**进行校正：

```
并行启动 N 个子代理（N = 章节数量，建议最多8个）:

子代理 1: 校正 chapters/chapter-01-xxx.md
子代理 2: 校正 chapters/chapter-02-xxx.md
...
子代理 N: 校正 chapters/chapter-N-xxx.md
```

**子代理配置**：
- 使用 `general-purpose` agent 类型
- 每个子代理独立处理一个章节
- 子代理用 Edit 工具直接修改原文件
- 修复后保留原文件结构（YAML frontmatter、标题层级、段落格式）

### Step 4: 生成校正报告

所有子代理完成后，在主会话中生成校正报告：

```markdown
# OCR 校正报告 — {书名}

## 统计
- 校正文件数: {N}
- 总修复处数: {M}
- 主要问题类型分布:
  - 乱码片段: X 处
  - 断裂句子: Y 处
  - 多余空行: Z 处
  - 页码残留: W 处
  - OCR 错字: V 处

## 各章节详情

### chapter-01-xxx.md
- 修复: {数量} 处
- 主要问题: [乱码/断裂/错字/...]
- 示例:
  - 修复前: `...`
  - 修复后: `...`

...
```

## 需要修复的典型 OCR 问题

### 1. 乱码片段

孤立的无意义字符组合，通常是 OCR 识别失败的结果：

```
GB wenwor
a7
40 一
— 123 —
■ □ ○
```

### 2. 断裂的句子

OCR 将行尾换行错误地保留，导致一句话被截断成多行：

```
在这种意义上，意识就是某些现象品质的例示。这是"意识"术
语的关键意义，或者至少是形成主要的解释问题的一种意义。
```

→ 应修复为连续段落：
```
在这种意义上，意识就是某些现象品质的例示。这是"意识"术语的关键意义，或者至少是形成主要的解释问题的一种意义。
```

### 3. 多余空行

连续3个以上的空行，通常是页面分页残留：

```
段落A




段落B
```

→ 最多保留2个空行：
```
段落A

段落B
```

### 4. 页码残留

页码和页眉/页脚信息混入正文：

```
40 一
— 123 —
第 45 页
```

### 5. OCR 错字

常见 OCR 识别错误（中文）：

| 错误 | 正确 | 说明 |
|------|------|------|
| 氨今 | 迄今 | 形近字误识 |
| 错情 | 错愕 | 上下文语义错误 |
| 附随性 | 附随性 | 正确（非错误） |
| 心身 | 心身 | 正确（非错误） |

**修复原则**：
- 仅修复**明显是 OCR 错误**的字（乱码、无意义组合、上下文不通）
- 不修改专业术语（即使看起来陌生）
- 不修改人名、地名等特殊名词
- 保持原文意思不变，不做内容增删

## 子代理提示词模板

```
请校正以下 Markdown 文件中的 OCR 排版错误。

文件路径: {chapter-file-path}

## 修复清单

1. **删除乱码片段**：删除明显无意义的孤立字符（如 "GB wenwor"、"a7"、"40 一"、纯符号行）
2. **修复断裂句子**：将错误换行截断的句子重新连接为一行，保持段落连贯
3. **合并多余空行**：连续3个以上空行合并为最多2个
4. **删除页码残留**：删除混入正文的页码（如 "— 123 —"、"40 一"）
5. **修复 OCR 错字**：修复明显的识别错误（如 "氨今" → "迄今"），但**不要修改专业术语**
6. **保留格式**：保留 YAML frontmatter、标题层级（# ## ###）、列表、引用块等 Markdown 格式

## 重要原则

- 只修复 OCR 导致的错误，不做内容改写
- 专业术语保持原样，即使你不认识
- 不确定的地方不要修改
- 用 Edit 工具直接修改文件，不要生成新文件
```

## 注意事项

- 书籍路径必须是绝对路径
- 校正直接修改 `chapters/` 下的源文件，**不会备份**
- 建议在运行前用 Git 提交，以便必要时回滚
- 并行子代理数量受限于系统资源，建议最多8个
- 如果某章内容过长（>50KB），子代理可能需要更长处理时间
- 校正后的文件可直接用于后续的 `/deep-reader` 或 `/book-reader` 流程
