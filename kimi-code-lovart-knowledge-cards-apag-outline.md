## 2. Perspective（视角 / 问题）

> 目标：让读者感到「这说的就是我」，而不是立刻给答案。

### 2.1 读者当前错误信念

- 「知识卡片 = 设计能力，我不会 PS/AI 就做不了。」
- 「内容好就行，视觉只是锦上添花。」
- 「做一张卡片要一小时，批量做不现实。」
- 「AI 生成的图风格不稳定，没法形成系列。」
- 「卡片就是文字排版，截图发出去也能看。」

### 2.2 这些信念带来的痛苦

- 笔记越积越多，能发出去的越来越少。
- 花了两小时排版，发出去没人看。
- 想做一个系列卡片，但每张风格都不一样。
- 看到别人的卡片好看，自己一做就廉价。
- 每次做图都从零开始写提示词，效率极低。

### 2.3 常见失败模式

- 用 PPT/Canva 手动调，重复劳动。
- 直接截图文字，毫无设计感。
- 生图提示词写不准，出来的图和主题无关。
- 没有统一风格，账号像杂货铺。
- 内容和视觉脱节，卡片「好看但没说清」。

### 2.4 场景化描写

「你周末整理了一篇 2000 字的读书笔记，想做成 5 张卡片发到小红书。你打开设计软件，选模板、调字体、找配图，两个小时过去，只做了 1 张，而且还觉得不够好看。最后你干脆把文字截图发了出去，点赞 3 个。」

## 3. Advantage（优势 / 更好的视角）

> 目标：让读者相信存在更好的做事方式。

### 3.1 核心新观点

知识卡片不是设计作品，而是「结构化的视觉产品」；用 Kimi Code 负责结构与文案，Lovart 负责视觉风格，就能把卡片生产变成可重复的流水线。

### 3.2 之前-之后对比

| 之前（旧视角） | 之后（新视角） |
|----------------|----------------|
| 手动排版，1 小时 1 张 | AI 生成，5 分钟 1 张 |
| 风格凭感觉，每张都不一样 | 一套提示词锁定一套视觉系统 |
| 笔记和发布脱节 | Markdown 笔记直接转卡片 |
| 每张图从零描述 | 结构化 JSON 直接喂给生图工具 |
| 一个人闷头设计 | AI 代理 + 生图模型分工协作 |

### 3.3 支撑证据

- **社会证明**：越来越多 AI 内容创作者用「结构化文案 + 批量生图」工作流日更知识卡片。
- **权威引用**：可引用 Kimi Code 的 agentic 协作能力与 Lovart 的图像生成能力作为工具背书。
- **个人经验**：作者用这条链路做过读书笔记、概念卡片、金句海报等多种风格卡片。
- **类比/隐喻**：就像乐高积木——Kimi Code 拼出积木规格，Lovart 负责喷漆上色；规格统一，风格多变。

### 3.4 关键概念

- **xl-card**：结构化文案卡片 JSON，让生图工具可直接解析渲染。
- **视觉系统**：固定的风格提示词 + 版式约束，保证系列卡片风格统一。
- **知识卡片流水线**：内容 → 结构化 → 风格化 → 批量输出。
- **风格锚点**：用具体艺术家、设计风格、场景、情绪词组合来锁定视觉方向。

## 4. Amplify/Gamify（放大 / 行动阶梯）

> 目标：把认知转变变成可执行步骤。

### 4.1 转变路径

```
笔记堆积 → 结构化文案 → 定义视觉系统 → 批量生图 → 建立素材库
```

### 4.2 分步行动清单

#### 步骤 1：用 deep-reader 对文章做知识点拆解

- **做什么**：把目标文章喂给 deep-reader，让它提取并结构化所有核心知识点。
- **即时收益**：得到一批清晰、独立、可直接复用的知识点，而不是整篇长文。
- **常见障碍**：文章太长，一次拆不完；知识点边界模糊。
- **小贴士**：先让 deep-reader 输出「概念 + 定义 + 应用场景」三件套，方便后续转卡片。

#### 步骤 2：用 xl-card 把知识点转成 JSON 卡片

- **做什么**：基于 deep-reader 输出的知识点，调用 xl-card skill 生成结构化的卡片 JSON。
- **即时收益**：JSON 结构稳定，换风格时不会丢失内容，可批量量产。
- **常见障碍**：字段不统一，导致后续渲染出错。
- **小贴士**：固定一套字段模板（如标题、核心观点、支撑要点、金句、视觉关键词），所有卡片都按同一套结构走。

#### 步骤 3：在 Pinterest / 花瓣找到你喜欢的风格图

- **做什么**：去 Pinterest、花瓣等视觉网站搜索你喜欢的知识卡片风格，保存参考图。
- **即时收益**：有了明确的视觉方向，出图不再靠运气。
- **常见障碍**：风格太多，不知道选哪个；收藏了但用不上。
- **小贴士**：一次只锁定 1-2 种风格作为「风格锚点」，先把一套跑通再扩展。

#### 步骤 4：在 Lovart 中上传参考图并粘贴 JSON 卡片

- **做什么**：打开 Lovart，使用 nano banana 模型，上传步骤 3 找到的参考图，再把 xl-card 生成的 JSON 文件复制进去。
- **即时收益**：Lovart 会基于参考图风格和 JSON 内容生成对应的知识卡片。
- **常见障碍**：参考图和 JSON 内容不匹配；第一次出图需要微调。
- **小贴士**：如果参考图风格偏复杂，可以先用一张做测试，确认效果后再批量跑。

#### 步骤 5：建议购买 Lovart 会员实现无限制生图

- **做什么**：如果你计划持续做知识卡片，建议购买 Lovart 会员。
- **即时收益**：无限制生图，批量出图时不心疼，方便迭代风格和内容。
- **常见障碍**：担心投入产出比。
- **小贴士**：先按免费额度跑一轮验证效果，确认适合自己后再开通会员。

### 4.3 行动号召（CTA）

「现在就挑一篇你最近读过的文章，走一遍这个流程：deep-reader 拆知识点 → xl-card 转 JSON → Pinterest 找风格 → Lovart 出图。30 分钟后，你就会看到自己的第一张 AI 知识卡片。」
